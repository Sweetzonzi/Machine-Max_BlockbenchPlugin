# JSON Schema在Machine-Max内容包制作中的应用指南

## 什么是JSON Schema？

JSON Schema是一种用于描述和验证JSON文档结构的规范。对于Machine-Max的内容包制作，JSON Schema能够帮助你：

- **验证配置格式**：确保你的JSON配置文件格式正确
- **提供智能提示**：在支持的编辑器中显示属性说明和类型要求
- **减少错误**：提前发现配置错误，避免游戏内问题

## 为什么内容包制作者需要关注JSON Schema？

即使你没有编程经验，JSON Schema也能让你的内容包制作过程更顺畅：

1. **自动补全**：现代编辑器（如VSCode）可以根据Schema提供属性名自动补全
2. **错误提示**：在配置错误时立即提示，而不是等到游戏中才发现问题
3. **属性说明**：查看每个配置项的具体含义和使用方法
4. **类型检查**：确保你输入的数据类型正确（数字、字符串、布尔值等）

## 如何使用JSON Schema？

### 1. 配置编辑器支持

在你的JSON文件顶部添加`$schema`字段，或直接复制已有的示例JSON，例如：

```json
{
  "$schema": "../../docs/zh_cn/schemas/part_definition_schema.json",
  "vehicle_durability_rate": 1.0,
  "variants": {
    "default": {
      "icon": "machine_max:textures/icon/example.png",
      "models": "machine_max:example.geo"
    }
  }
}
```
注意，在这里`$schema`字段的值为相对路径，"../"表示当前文件的上级目录。

### 2. 推荐编辑器

- **VSCode**：原生支持JSON Schema

### 3. 在VSCode中启用JSON Schema验证

 **推荐直接复用官方内容包的文件结构，并原样复制docs文件夹到你自己的内容包，再在部件和子系统json文件开头添加`$schema`引用即可。**

## 实际制作中的应用示例

### 步骤1：创建新部件

1. 复制现有的部件JSON文件作为模板
2. 在文件开头添加`$schema`引用
3. 根据编辑器的智能提示添加配置项

### 步骤2：验证配置

1. 保存文件后查看编辑器是否显示错误
2. 根据提示修正配置格式问题
3. 测试内容包在游戏中是否正常工作

### 步骤3：调试问题

如果内容包在游戏中不工作，检查：

1. JSON格式是否正确（使用Schema验证）
2. 所有必需字段是否都已填写
3. 数据类型是否匹配（数字 vs 字符串等）

## 常见问题与解决方案

### Q: 编辑器提示错误，但配置看起来是正确的？
A: 检查：
- JSON语法是否正确（逗号、括号是否有缺漏等，务必注意不要使用中文标点符号）
- 字段名称是否拼写正确
- 数据类型是否匹配（如需要数字但输入了字符串）

### Q: 如何知道哪些字段是必需的？
A: 查看Schema文件中的`required`字段，或者在支持的编辑器中会有相应提示。

### Q: 这些字段的作用都是什么？
A: 将鼠标放到字段上，即可查看字段的说明和类型要求。

## 提高效率的技巧

1. **使用模板**：基于现有工作正常的配置创建新配置
2. **逐步添加**：先添加基本配置，验证无误后再添加高级选项
3. **参考示例**：查看官方包中的示例配置文件
4. **实时验证**：在编辑器中实时查看验证结果

## 总结

JSON Schema是内容包制作的强大辅助工具，它能帮助你创建格式正确、结构完整的配置文件。即使没有编程经验，使用支持Schema的编辑器也能大大提高制作效率和准确性。

开始使用时，建议先从简单的配置开始，逐步熟悉Schema提供的各种功能，随着经验的积累，你会发现制作内容包变得更加简单和可靠。
