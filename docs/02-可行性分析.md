## 3. 风险点详解

### 3.1 ⚠️ `.bbmodel` extra 字段持久化

**风险**：Blockbench 的 `.bbmodel` 保存/加载逻辑可能会清理或覆盖非标准字段。

**验证方法**：
1. 手动在 `.bbmodel` 的 JSON 根添加 `"machine_max": {...}` 字段
2. 在 Blockbench 中打开后立即保存
3. 检查保存后的文件中该字段是否保留

**缓解方案**：如果 `.bbmodel` 不支持，则使用独立配置文件。插件自动在同目录创建 `.bbmodel.mm_config.json`。

**实现注意**：`.bbmodel` 的 JSON 根节点下，Blockbench 通常保留未知字段（不做清理），但需要在 Blockbench 中验证。若 BB 丢弃额外字段，备选方案为独立 `.mm_project.json` 文件。

---

### 3.2 ⚠️ Outliner 右键菜单扩展

**风险**：`OutlinerNode.showContextMenu(event)` 的菜单结构是硬编码的，没有提供 `addItem` 这类扩展 API。

**解决思路**：
- **方案A（拦截法）**：用 `Proxy` 包装 `showContextMenu`，在菜单弹出前插入自定义项
- **方案B（事件法）**：在 Outliner 上添加独立的 `contextmenu` 事件监听，`preventDefault` 后完全自定义菜单
- **方案C（推荐）**：在模式进入时替换相关 Group/Locator 的 `showContextMenu` 方法，调用原始方法获取菜单结构后追加自定义项，模式退出时恢复

---

### 3.3 ⚠️ Vue 组件在 Mode 中的生命周期

**风险**：Mode 的 `component` 选项需要 Vue 组件构造函数，且需确保 Mode 切换时组件正确挂载/销毁。

**说明**：Blockbench 内部使用 Vue 2。`Mode` 构造函数中检查 `data.component`，自动创建 `#center` 下的挂载点并实例化 Vue（见 `modes.ts:L58-L76`）。这与 Dialog 的 `component` 机制类似。

**需要注意**：
- 组件销毁时清理事件监听（`beforeDestroy`）
- 多个模式之间切换时确保组件不残留

## 5. 不建议使用的方式

| 方式 | 原因 |
|------|------|
| 修改 Blockbench 核心源码 | 需要维护分支，升级困难 |
| 通过 `eval` 或 `Function` 动态执行 | 安全风险，且不需要 |
| Web 版文件操作 | Web 版无法访问本地文件系统，本项目只支持桌面版 |
| 依赖外部构建工具 (webpack/vite) | Blockbench 插件不需要构建，直接用 `<script>` 标签加载即可 |
