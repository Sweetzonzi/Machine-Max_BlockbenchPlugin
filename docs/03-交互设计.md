# 03 — 交互设计 (修订版)

> 最后更新：2026-05-18 | 修订：新增信号路由表单 UI 设计（Phase 1）+ 信号拓扑预览（Phase 2）+ 交互式蓝图（Phase 3）

---

## 1. 设计目标

- **零命名记忆**：所有骨骼/Locator 通过可视化选择，不要求用户记忆命名规则
- **所见即所得**：3D 视口中用颜色标注当前变体的标记
- **渐进复杂度**：简单场景默认值覆盖，高级参数折叠隐藏
- **上下文驱动**：根据当前选中元素类型和层级，自动切换右侧属性面板内容
- **多零件管理**：一个模型可定义多个零件，每个零件有独立变体

## 3. 项目级管理

### 3.1 零件切换

顶部的导航栏是**模式内的核心控件**：

```
零件: [wine_fox_hull      ▾]  变体: [left ▾]  [+ 新建零件]  [项目管理]
```

| 控件 | 行为 |
|------|------|
| `零件` 下拉 | 切换当前编辑的零件 → Outliner 图标、3D 标注、右侧面板全部刷新 |
| `变体` 下拉 | 切换当前编辑的变体 → 同零件内切换变体视图 |
| `[+ 新建零件]` | 弹出对话框：输入零件 ID（如 `wine_fox_gt_spoiler`）→ 创建空零件配置 |
| `[⚙ 项目管理]` | 打开项目全貌面板（见 3.3） |

### 3.2 切换流程

```
用户切换零件 "wine_fox_door"
        ↓
1. 保存当前零件 "wine_fox_hull" 的修改
2. 清除所有 Outliner 图标和 3D 标注
3. 加载 "wine_fox_door" 的 element_markers
4. 读取当前变体（上次退出时记录，或第一个变体）
5. 仅高亮 currentVariant 的标记：
   → Outliner 图标更新
   → 3D 标注创建
6. 右侧面板刷新 → 显示空闲页（零件全局属性）
        ↓
用户切换变体 "right"
        ↓
1. 保存当前变体 "left" 的修改
2. 清除标注
3. 加载 "right" 的 element_markers
4. 刷新标注
5. 面板回到空闲页
```

## 4. Outliner 增强

### 4.1 图标体系

进入零件定义模式后，**仅当前零件当前变体的已标记元素**更换图标/颜色。未标记元素和不在当前变体范围内的元素保持默认外观。

| 用途 | 图标 | Font Awesome class | 颜色标记 |
|------|------|-------------------|---------|
| 未标记 (默认) | 默认图标 | 不作修改 | 灰色 |
| 子零件 Group | 📦 `fa-box` / `fa-cube` | `fa fa-cube` | 蓝色 #4A90D9 |
| 碰撞箱 Group | 🔵 `fa-shield` | `fa fa-shield` | 红色 #D94A4A |
| 连接点 Locator | 📌 `fa-plug` | `fa fa-plug` | 绿色 #4AD94A |


> 子系统**不在 Outliner 中有对应节点**，子系统实例作为子零件面板中的纯数据条目存在，由子零件面板的列表统一管理。详见 §6.6。

### 4.2 右键菜单

进入模式后，Outliner 中各节点的右键菜单追加 MachineMax 专属项：

**Group（骨骼）右键菜单追加：**
```
┌──────────────────────────┐
│ (原有菜单项...)          │
│ ────────────────────────│
│ inventory_2  标记为子零件│  ← 新增
│ select_all   标记为碰撞箱│  ← 新增
│ add          添加子系统...│  ← 新增（弹出类型选择对话框选择20种子系统类型）
│ ────────────────────────│
│ delete        清除标记   │  ← 新增（仅已标记时显示）
└──────────────────────────┘
```

**Locator（定位器）右键菜单追加：**
```
┌──────────────────────────┐
│ (原有菜单项...)          │
│ ────────────────────────│
│ link  标记为连接点       │  ← 新增
│ ────────────────────────│
│ delete  清除标记          │  ← 新增（仅已标记时显示）
└──────────────────────────┘
```

### 4.3 快速筛选

Outliner 顶部增加筛选按钮栏（仅在零件定义模式下显示）：

```
[全部] [子零件] [碰撞箱] [连接点] [子系统] [未标记]
```

作用：过滤 Outliner 只显示对应用途的节点，帮助作者快速定位。

## 5. 3D 视口标注

### 5.1 颜色高亮体系

在 3D 视口中，**仅当前零件当前变体的已标记元素**显示颜色标注：

| 用途 | 3D 表现 | 实现方式 |
|------|---------|---------|
| 子零件骨骼 | **蓝色** 半透明 BoxHelper | THREE.BoxHelper |
| 碰撞箱骨骼 | **红色** 半透明 BoxHelper | THREE.BoxHelper |
| 碰撞箱方块 | **红色** 线框覆盖 (Wireframe) | THREE.LineSegments |
| 连接点 Locator | **绿色** Sprite + 方向箭头 | THREE.Sprite + THREE.ArrowHelper |
| 未标记骨骼 | **灰色** 半透明 BoxHelper | THREE.BoxHelper |

> 子系统不是 3D 元素，在子零件面板中以列表形式配置，不参与 3D 视口标注。详见 §6.6。

### 5.2 标注生命周期

- **进入模式**：遍历所有标记配置，创建对应 THREE 对象加入 `Canvas.gizmos`
- **标记变更**：即时更新（删除旧标注 + 创建新标注）
- **退出模式**：批量清除所有 gizmos
- **相机变化**：Sprite 始终面向相机（THREE.Sprite 自动处理）

### 5.3 悬停高亮

鼠标悬停在 Outliner 中的节点上时：
- 对应 3D 标注高亮（增大亮度或改变材质）
- 显示 Tooltip（用途名称 + 简要配置摘要）

## 6. 右侧属性面板

### 6.1 面板概述

属性面板根据**当前编辑层级**动态切换内容。层级关系：

```
零件全局 (PartType级)  →  变体属性 (VariantAttr级)  →  选中元素详情
```

用户在不同场景下看到不同页面：

**面板状态机：**

```

│ 无选中   │ ──────────────────→  │ 子零件属性页 │
```

### 6.8 名称与骨骼分离的设计总结

本插件遵循**名称（字典 key）** 与 **骨骼/Locator 绑定**分离的设计原则：

| 实体 | 字典 key（名称） | 绑定字段 | 默认值规则 |
|------|-----------------|---------|-----------|
| **子零件** | `sub_part.<ns>.main` | `start_bone` = 骨骼名 | 首次标记 Group 时生成 `sub_part.<ns>.main`（如需自增后缀去重） |
| **连接点** | `connector.<ns>.<名称>` | `locator` = Locator 名 | 首次标记 Locator 时，名称 = `connector.<ns>.<locator名>` |
| **子系统** | `subsystem.<ns>.<名称>` | `locator` / `connector`（均为可选，仅特定类型需要） | 新建时生成 `subsystem.<ns>.<type短名>_<序号>`，不绑定任何 Outliner 元素 |
| **碰撞箱** | **(key 本身就是骨骼名)** | 无需绑定字段 | 不变，key = 骨骼名（与内容包一致） |
| **交互区** | `interact.<ns>.<名称>` | `bone` = 骨骼名 | 不变，已有独立 bone 字段（与内容包一致） |

**关于子系统的说明：**

子系统**不在 Outliner 中有对应节点**，因此不存在"标记"操作。`locator` 和 `connector` 只是子系统实例的可选字段，通过面板内的下拉选择器配置。具体哪些类型需要这些字段由 `subsystem_types.js` 中的 `needsLocator` / `needsConnector` 元数据控制：

| 可选的引用字段 | 需要该字段的子系统类型 | 用途 |
|---------------|----------------------|------|
| `locator` | `seat`, `launcher`, `turret`, `fire_controller`, `camera` | 空间定位点（乘坐位置、发射点、喷口位置等） |
| `connector` | `wheel_driver`, `joint`, `launcher`, `turret`, `fire_controller` | 对连接点的控制引用 |
| 均不需要 | `engine`, `gearbox`, `car_controller`, `transmission`, `motor`, `basic`, `item_storage`, `lighting`, `motor_controller`, `battery`, `signal_convert`, `javascript` | 纯控制/计算逻辑，通过信号路由通信 |

**命名约束矩阵：**

| 作用域 | 约束 | 示例 |
|--------|------|------|
| 子零件名称 | 同一变体内**唯一** | 变体 `left` 中不能有 2 个 `sub_part.machine_max.main` |
| 连接点名称 | 同一子零件内**唯一** | 子零件 `main` 中不能有 2 个 `connector.machine_max.xxx` |
| 子系统名称 | 同一子零件内**唯一** | 子零件 `main` 中不能有 2 个 `subsystem.machine_max.engine` |
| 碰撞箱 key | 同一子零件内**唯一** | 现有规则，不变 |
| 交互区 key | 同一子零件内**唯一** | 现有规则，不变 |

**默认值中的 `<ns>`** 从零件所属的内容包命名空间获取，通常为 `machine_max`。

**编辑名称时的同步行为：**
1. 用户修改名称（即字典 key）
2. 系统执行 `this.$set(variant.sub_parts, 新名称, 配置数据)` + `this.$delete(variant.sub_parts, 旧名称)` 迁移字典
3. 更新 `element_markers` 中对应的 `config_ref`（子零件/连接点）
4. 重算 `auto_end_bones`（仅子零件）
5. 刷新 Outliner 图标
6. 信号路由中的引用暂不自动更新，后续版本提供"查找引用"功能

## 8. 交互式蓝图编辑器（Phase 3 交互设计）

### 8.1 定位

Phase 3 在 Phase 2 的只读拓扑图基础上增加**拖拽连线编辑**能力。蓝图编辑器和表单形成双向绑定：编辑蓝图 → 自动更新表单 JSON；编辑表单 → 蓝图刷新。

### 8.2 布局

蓝图编辑器取代只读拓扑图，使用同一面板位置但交互模式不同：

```
┌─ 信号蓝图: ae86_chassis (default) ──────┐
│                                          │
│  [自动布局] [保存布局] [重置] [缩放: 100%]│
│ ┌──────────────────────────────────────┐ │
│ │   (输出)         (信号转发)   (输入)  │ │
│ │                                      │ │
│ │  seat    ─move_input→ controller   │ │
│ │  sub.   ─regular_input→ ctrl.     │ │
│ │  seat   ────interact────▶ seat sub. │ │
│ │   ▲                     │            │ │
│ │   │        engine_speed │            │ │
│ │   └──────── engine ◀────┘            │ │
│ │              │                       │ │
│ │         power│                       │ │
│ │              ▼                       │ │
│ │            gearbox                   │ │
│ │                                      │ │
│ │  右键连线 → 删除   │  拖拽端口 → 连线  │ │
│ └──────────────────────────────────────┘ │
│ [保存并关闭]                              │
└──────────────────────────────────────────┘
```

### 8.3 新增交互能力

| 交互 | 操作方式 | 结果 |
|------|---------|------|
| **创建连线** | 从节点右侧（输出端口）拖拽到左侧（输入端口） | 自动创建 `signal_targets` 条目 |
| **删除连线** | 右键点击连线 → "删除该路由" | 移除对应的 `signal_targets` 条目 |
| **添加节点** | 空白区域右键 → "添加信号源/目标" | 在下拉列表中选择要添加的实体 |
| **拖动节点** | 拖拽节点标题区域 | 改变节点位置（用于自定义布局） |
| **编辑频道名** | 双击连线标签 | 弹出频道名编辑输入框 |
| **添加转译** | 右键节点 → "添加信号转译" | 在 `signal_translations` 中添加条目 |
| **缩放** | Ctrl+滚轮 | 画布缩放（0.5x ~ 2.0x） |

### 8.4 连线绘制方式

采用**正交折线**（Orthogonal Polyline）绘制连线，而非动画控制器中的 CSS 拱形：

```
                  ┌──────────────┐
 controller ──────┤              │──────→ wheel_driver
                  │  正交折线路径  │
 seat ────────────┤              │──────→ engine
                  │  (避开其他节点)│
                  └──────────────┘
```

正交折线自动路由规则：
- 先从源端口水平引出，再垂直转向，最后水平接入目标端口
- 遇到中间节点时自动绕行（预留间距）
- 多条连线并行时分层错开

### 8.5 双向同步机制

| 方向 | 时机 | 动作 |
|------|------|------|
| 蓝图 → 表单 | 拖拽连线/删除连线/编辑频道名完成时 | 立即更新 `Project.machine_max_plugin` 中的对应字段 |
| 表单 → 蓝图 | 表单输入变化时 | 当前零件变体的信号拓扑重新计算，蓝图重绘 |
| 保存/切换 | 切换零件/变体或退出模式时 | 蓝图当前布局位置存入持久化（非必须，自动布局可恢复） |

## 12. 与原有 Blockbench 功能的协作

| 原有功能 | 在零件定义模式下的行为 |
|---------|----------------------|
| 建模工具 (move/rotate/scale) | **禁用**。3D 视口变为只读 |
| 撤销/重做 | 对 MachineMax 配置的修改应走自己的撤销栈 |
| 贴图编辑 | **不可用**。切换到 Paint 模式使用 |
| 动画编辑 | **不可用**。切换到 Animate 模式使用 |
| 文件保存 (Ctrl+S) | 保存 .bbmodel 文件（含 MachineMax 配置） |

## 13. 可访问性

- **键盘快捷键**：
  - `Ctrl+Shift+P` → 切换零件定义模式
  - `Ctrl+Shift+E` → 导出内容包
  - `Ctrl+Shift+V` → 校验配置
  - `Tab` → 在属性面板字段间切换
- **错误反馈**：表单字段输入无效时，字段边框变红 + Tooltip 显示错误信息
- **进度反馈**：导出时显示进度条
- **撤销支持**：配置修改支持 Ctrl+Z 撤销（独立撤销栈）
