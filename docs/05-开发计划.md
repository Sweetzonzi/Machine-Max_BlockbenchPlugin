# 05 — 开发计划

> 最后更新：2026-05-14

---

## 1. 开发策略

采用**分阶段交付**策略，每个阶段产出可独立使用的功能：

```
阶段一 ──→ 阶段二 ──→ 阶段三 ──→ 阶段四 ──→ 阶段五
(骨架)     (连接点)   (子系统)   (导出)     (增强)
```

每阶段完成后即可交付给 UGC 作者试用并提供反馈。

---

## 2. 阶段总览

| 阶段 | 名称 | 预估工作量 | 核心交付物 | 可独立使用？ |
|------|------|-----------|-----------|------------|
| 一 | 基础框架 | 中等 | 模式注册 + Outliner 增强 + 零件全局配置 | ✅ 可搭配手工 JSON |
| 二 | 连接点与碰撞箱 | 中等 | 可视化标记 + 属性面板 + 静态定义编辑 | ✅ 核心价值 |
| 三 | 子系统配置 | 较大 | 18种子系统类型的动态表单 + 信号路由 | ✅ 完整覆盖 |
| 四 | 导出与持久化 | 中等 | 多文件导出 + .bbmodel 持久化 + 校验 | ✅ 完整体验 |
| 五 | 增强功能 | 较小 | 材料编辑器 + 自动分析 + 本地化 + 优化 | — |

## 5. 阶段三：子系统配置

### 5.1 目标

实现 18 种子系统类型的可视化配置，包括型号编辑和信号路由。

### 5.2 任务列表

MachineMax 共定义了 **19 种**子系统类型（12 种已实装 + 7 种测试中/未实装，详见 [01-总体设计 §7.2](./01-总体设计.md#72-支持的子系统类型)）。本阶段建议优先实现已实装的 12 种，测试中的类型待模组正式发布后跟进。

**第一批（优先/已实装，6 种）：**
| 类型 | 用途 | 复杂度 |
|------|------|--------|
| `engine` | 发动机 | 中 |
| `wheel_driver` | 轮胎驱动 | 中 |
| `seat` | 座位 | 低 |
| `gearbox` | 变速箱 | 低 |
| `car_controller` | 车辆控制器 | 中 |
| `motor` | 电动机 | 低 |

**第二批（已实装，6 种）：** `basic`, `item_storage`, `lighting`, `motorbike_controller`, `transmission`, `motor_controller`

**第三批（测试中/待跟进，7 种）：** `battery`, `joint`, `signal_convert`, `camera`, `javascript`, `fire_controller`, `turret`, `launcher`

> **注意**：第三批中的 `fire_controller`、`turret`、`launcher` 目前在 MachineMax 模组中处于测试阶段尚未实装，本插件将在其正式发布后跟进支持。

| 编号 | 任务 | 描述 | 依赖 |
|------|------|------|------|
| 3.1 | 子系统类型注册表 | 创建 `core/subsystem_types.js` → 定义 18 种类型的元数据（名称/图标/字段列表/默认值） | 1.3 |
| 3.2 | 右键菜单 → "添加子系统"子菜单 | 展开 18 种类型的可选列表 | 1.5, 3.1 |
| 3.3 | 属性面板 → 子系统属性页（动态模板） | Vue 组件：根据 `type` 动态渲染对应字段组 | 1.8, 3.1 |
| 3.4 | 新建子系统型号对话框（动态表单） | `new Dialog({form: ...})` → 根据 type 动态生成 `onFormChange` 切换字段 | 3.1, 3.3 |
| 3.5 | 子系统信号输出编辑器 | 动态键值对编辑（`power_output` / `speed_outputs` 等） | 3.3 |
| 3.6 | 第一批 6 种类型实现 | engine/wheel_driver/seat/gearbox/car_controller/motor 的完整字段定义（基于 Schema） | 3.1~3.5 |

| 3.9 | 3D 标注 → 子系统定位器 | 根据子系统类型显示不同颜色/图标（座位=黄色/灯光=橙色/通用=紫色） | 2.1, 3.1 |

## 8. 文件结构规划

开发完成后的 src 目录结构：

```
src/
├── plugin.js                    # 插件入口：Plugin.register()
├── mode.js                      # Mode 注册 + 进入/退出回调
│
├── core/
│   ├── config.js                # MMConfig 数据模型（CRUD + 序列化/反序列化）
│   ├── config_defaults.js       # 所有字段的默认值定义
│   ├── element_markers.js       # element_markers 管理（标记/取消/查询）
│   ├── annotation.js            # 3D 标注系统（创建/更新/清除）
│   ├── validator.js             # 配置校验器
│   └── subsystem_types.js       # 18 种子系统类型的元数据注册表
│
├── generators/
│   ├── part_generator.js        # MMConfig → parts/*.json
│   ├── connector_generator.js   # connector_defs → connectors/*.json
│   ├── subsystem_generator.js   # subsystem_defs → subsystems/*.json
│   ├── material_generator.js    # material_defs → materials/*.json
│   ├── lang_generator.js        # → lang/*.json
│   └── meta_generator.js        # → meta.json
│
├── importers/
│   ├── part_importer.js         # 从 parts/*.json 反向解析回填
│   ├── connector_importer.js    # 从 connectors/*.json 反向解析回填
│   └── subsystem_importer.js    # 从 subsystems/*.json 反向解析回填
│
├── ui/
│   ├── App.vue.js               # 主 Vue 组件（mode.component）
│   ├── panels/
│   │   ├── PartGlobalPanel.vue.js     # 空闲页：零件全局属性
│   │   ├── SubPartPanel.vue.js        # 子零件属性页
│   │   ├── HitBoxPanel.vue.js         # 碰撞箱属性页
│   │   ├── ConnectorPanel.vue.js      # 连接点属性页
│   │   └── SubsystemPanel.vue.js      # 子系统属性页
│   ├── dialogs/
│   │   ├── ConnectorDefDialog.js      # 新建连接点静态定义
│   │   ├── SubsystemDefDialog.js      # 新建子系统型号
│   │   ├── MaterialDefDialog.js       # 新建/编辑材料
│   │   ├── ExportDialog.js            # 导出内容包
│   │   └── ValidationDialog.js        # 校验结果
│   ├── forms/
│   │   ├── connector_form.js          # 连接点通用表单字段
│   │   ├── subsystem_forms.js         # 各子系统类型的表单字段
│   │   └── signal_target_editor.js    # 信号目标键值对编辑器
│   └── components/
│       ├── VariantManager.vue.js      # 变体管理组件
│       ├── TagEditor.vue.js           # 标签编辑器
│       ├── KeyValueEditor.vue.js      # 通用键值对编辑器
│       └── FoldableSection.vue.js     # 可折叠区域
│
├── utils/
│   ├── file_writer.js           # Node.js fs 封装（创建目录、写入文件）
│   ├── naming.js                # 命名工具（生成 connector.xxx / subsystem.xxx 名称）
```
