# I/O 与格式系统 API 文档

## 概述

Blockbench 的 I/O 与格式系统由三大部分组成：**模型格式（ModelFormat）**、**编解码器（Codec）** 和**全局 I/O 函数**。格式定义了模型编辑环境的能力边界，编解码器则负责将模型数据序列化为文件（编译）和从文件恢复（解析）。

---

## 1. ModelFormat 类 — 模型格式

表示 Blockbench 支持的一种模型格式。通过 Property 机制管理所有格式特性。

### 全局注册

```typescript
Formats: Record<string, ModelFormat>  // 所有已注册格式的集合
Format: ModelFormat                   // 当前激活的格式（全局只读引用）
```

### 构造

通常不直接通过 `new` 构造，而是通过 `Formats[id]` 访问已注册格式。插件可通过 `new ModelFormat(id, options)` 注册新格式：

```typescript
new ModelFormat('my_format', {
    id: 'my_format',
    icon: 'my_icon',
    name: '我的格式',
    description: '格式描述',
    category: 'general',
    show_on_start_screen: true,
    // ... 功能特性属性
    box_uv: false,
    bone_rig: true,
    rotate_cubes: false,
    codec: myCodec
})
```

### 方法

#### format.select()

激活该格式。将当前项目转换为此格式，更新网格、场景、预览、设置等，并派发 `select_format` 事件。

```javascript
Format.select()  // 激活当前格式
Formats.java_block.select()  // 切换到 Java 方块格式
```

**内部流程：**
1. 调用旧格式的 `onDeactivation`（如果存在）
2. 更新 `Blockbench.Format` 和 `Project.format` 引用
3. 重建网格和地面平面
4. 更新场景位置和地面缩放
5. 更新预览模型
6. 同步设置
7. 切换模式以匹配新格式的能力
8. 更新面板状态（动画面板、UV 编辑器等）
9. 调用新格式的 `onActivation`（如果存在）
10. 派发 `select_format` 事件

#### format.convertTo()

将当前项目转换为该格式。与 `select()` 不同，此方法会执行数据转换逻辑：
- 重置撤销历史
- 清空导出路径和未处理根字段
- 处理 Box UV 转换
- 处理纹理 UV 尺寸转换
- 处理骨骼绑定转换
- 处理网格中心偏移
- 处理纹理模式转换
- 处理节点名称清理
- 处理立方体旋转转换
- 移除不兼容的元素类型（网格、样条线、定位器等）
- 应用立方体尺寸限制和旋转限制
- 派发 `convert_format` 事件
- 调用 `onSetup` 回调

```javascript
Formats.java_block.convertTo()
```

#### format.new()

基于此格式创建新项目。调用 `newProject(this)` 内部函数，返回 `boolean` 表示成功与否。

```javascript
let success = Formats.java_block.new()
```

#### format.delete()

删除该格式。从 `Formats` 注册表中移除，清理关联的 codec 引用，派发 `delete_format` 事件。

```javascript
format.delete()
```

## 2. FormatFeatures 接口 — 格式功能特性

`FormatFeatures` 接口定义了格式所支持的全部功能。此接口与 `FormatOptions` 合并，作为 `ModelFormat` 构造函数的输入。所有属性均为可选的 boolean 或特定类型值，上面已完整列出。

### CubeSizeLimiter — 立方体尺寸限制器

当格式需要限制立方体大小时使用：

| 方法 | 说明 |
|------|------|
| `test(cube, values?)` | 测试立方体是否违反尺寸限制 |
| `move(cube, values?)` | 将立方体移回限制范围内 |
| `clamp(cube, values?, axis?, direction?)` | 将立方体夹紧到限制内 |
| `rotation_affected` | 旋转后是否检查和调整限制 |
| `coordinate_limits` | 可选的局部坐标限制 `[min, max]` |

### FormatPage — 启动屏幕页面配置

| 字段 | 类型 | 说明 |
|------|------|------|
| `component` | `Vue.Component` | Vue 组件 |
| `content` | `Array` | 内容数组，可包含图片、标题、文本等元素 |
| `button_text` | `string` | 按钮文本翻译键 |

## 3. Codec 类 — 编解码器

表示一种具体的文件格式，负责模型的序列化（编译）和反序列化（解析），以及加载和导出逻辑。继承自 `EventSystem` 和 `Deletable`。

### 全局注册

```typescript
Codecs: Record<string, Codec>  // 所有已注册编解码器的集合
```

### 构造

```javascript
var codec = new Codec('my_format', {
    name: '我的格式',
    extension: 'myfmt',
    load_filter: {
        extensions: ['myfmt', 'json'],
        type: 'json',
        condition: () => true
    },
    load: function(model, file, args) {
        // 加载逻辑：设置项目格式、解析模型等
    },
    parse: function(data, path, args) {
        // 解析逻辑：将 JSON 数据转换到 Blockbench 模型中
    },
    compile: function(options) {
        // 编译逻辑：将模型导出为字符串或 ArrayBuffer
        return JSON.stringify(result);
    },
    export: function() {
        // 导出逻辑：打开文件浏览器导出
    },
    export_options: {
        option1: { label: '选项1', type: 'checkbox', value: true }
    }
});
```

### CodecOptions 接口

| 字段 | 类型 | 说明 |
|------|------|------|
| `name` | `string` | **必需**。编解码器显示名称 |
| `load` | `(model, file, args?) => void` | 加载文件到程序。默认实现会调用 `this.parse()` 并设置项目，可覆盖 |
| `compile` | `(options?) => string \| ArrayBuffer \| any` | 编译模型内容。默认返回空字符串，并在编译前派发 `compile` 事件 |
| `parse` | `(data, path, args?) => void` | 解析文件内容并加载模型到 Blockbench 项目 |
| `export` | `() => void` | 打开文件浏览器导出。默认实现会提示导出选项、调用 `compile()`，然后通过 `Blockbench.export` 导出 |
| `merge` | `(model, args?) => void` | 将模型合并到当前项目中 |
| `fileName` | `() => string` | 建议的导出文件名。默认返回 `Project.name`（或 `'model'`） |
| `startPath` | `() => string` | 建议的起始路径。默认返回 `Project.export_path` |
| `write` | `(content, path) => void` | 写入文件。默认使用 `Blockbench.writeFile`，文件存在时调用 `overwrite` |
| `overwrite` | `(content, path, callback) => void` | 覆盖已存在文件的处理逻辑 |
| `afterDownload` | `(path) => void` | 下载/保存完成后的回调。默认更新 `Project.saved` 并显示提示消息 |
| `afterSave` | `(path) => void` | 保存完成后的回调。默认更新项目元数据、最近项目列表并显示提示消息 |
| `exportCollection` | `(collection) => void` | 导出集合的处理逻辑 |
| `writeCollection` | `(collection) => void` | 写入集合的处理逻辑 |
| `dispatchEvent` | `(event_name, data) => void` | 派发自定义事件 |
| `extension` | `string` | 默认文件扩展名（不含点号） |
| `remember` | `boolean` | 是否在最近文件列表中记住导出的模型 |
| `support_partial_export` | `boolean` | 是否支持通过集合部分导出 |
| `support_offset` | `boolean` | 是否支持偏移 |
| `load_filter` | `LoadFilter` | 加载过滤器，定义文件类型和扩展名匹配规则 |
| `export_options` | `Record<string, FormElementOptions>` | 导出选项表单配置 |
| `export_action` | `Action` | 关联的导出动作 |

#### LoadFilter 接口

| 字段 | 类型 | 说明 |
|------|------|------|
| `extensions` | `string[] \| (() => string[])` | 匹配的文件扩展名列表或返回列表的函数 |
| `type` | `'json' \| 'text' \| 'image'` | 文件内容类型 |
| `condition` | `ConditionResolvable` | 可选额外条件 |

### Codec 属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 编解码器 ID |
| `name` | `string` | 显示名称 |
| `extension` | `string` | 默认文件扩展名 |
| `remember` | `boolean` | 是否记住文件 |
| `support_partial_export` | `boolean` | 是否支持部分导出 |
| `support_offset` | `boolean` | 是否支持偏移 |
| `export_action` | `Action` | 导出动作 |
| `export_options` | `Record<string, FormElement>` | 导出选项表单元素 |
| `format` | `ModelFormat` | 关联的格式 |
| `plugin` | `string` | 创建该 codec 的插件 ID |
| `context` | `Collection \| null` | 当前上下文（集合导出时使用） |

### Codec 方法

#### codec.load(model, file, args?)

加载文件到程序中。默认实现：

1. 若 `!args.import_to_current_project`，创建新项目并切换到关联格式
2. 若文件路径有效且需要记住，设置项目名称、导出路径和 codec
3. 调用 `this.parse(model, file.path, args)`
4. 若需要记住，调用 `loadDataFromModelMemory()` 并添加到最近项目列表
5. 更新设置

```javascript
codec.load(jsonData, { path: 'model.bbmodel' })
codec.load(jsonData, file, { import_to_current_project: true })
```

#### codec.compile(options?)

编译模型内容。默认派发 `compile` 事件并返回空字符串。通常在子类中覆盖：

```javascript
let content = codec.compile()
let content = codec.compile({ raw: true, format: 'json' })
```

#### codec.parse(data, path, args?)

解析并加载模型。必须由具体 codec 实现：

```javascript
codec.parse(jsonObject, 'path/to/file.json', { import_to_current_project: true })
```

#### codec.export()

打开文件浏览器并执行导出。流程：

1. 如果定义了 `export_options`，调用 `promptExportOptions()` 提示用户输入
2. 调用 `Blockbench.export` 打开文件保存对话框
3. 通过 `compile(options)` 获取文件内容
4. 在桌面应用中通过 `custom_writer` 使用 `write()` 方法写入
5. 完成后调用 `afterDownload(path)`

```javascript
codec.export()
```

#### codec.fileName()

获取建议的导出文件名。若当前上下文为集合，返回集合名称；否则返回 `Project.name || 'model'`。

#### codec.startPath()

获取建议的起始路径。若当前上下文为集合，返回集合的 `export_path`；否则返回 `Project.export_path`。

#### codec.write(content, path)

写入文件到指定路径。若文件已存在且定义了 `overwrite` 方法，调用 `overwrite`；否则使用 `Blockbench.writeFile`。写入成功后调用 `afterSave(path)`。

#### codec.getExportOptions()

获取存储的导出选项值。合并 `export_options` 中各表单元素的当前值和 `Project.export_options` 中保存的值。

```javascript
let options = codec.getExportOptions()
```

#### codec.promptExportOptions()

弹出对话框提示用户输入导出选项。返回 `Promise`，解析为选项对象（用户确认时）或 `null`（用户取消时）。用户的选择会保存到 `Project.export_options` 中。

```javascript
let options = await codec.promptExportOptions()
```

#### codec.exportCollection(collection)

导出集合。临时修改上下文为集合，仅导出集合中的元素，完成后恢复：

```javascript
codec.exportCollection(myCollection)
```

#### codec.writeCollection(collection)

写入集合到文件。类似 `exportCollection` 但不打开文件浏览器：

```javascript
codec.writeCollection(myCollection)
```

#### codec.on(event_name, callback)

添加事件监听器。继承自 `EventSystem`：

```javascript
codec.on('compile', function(data) {
    console.log('编译中...', data)
})
```

#### codec.once(event_name, callback)

一次性事件监听：

```javascript
codec.once('parsed', function() {
    console.log('解析完成')
})
```

#### codec.delete()

删除编解码器。从 `Codecs` 注册表中移除，并清理关联格式的引用。

```javascript
codec.delete()
```

#### Codec.getAllExtensions() — 静态方法

获取所有已注册 codec 的全部扩展名列表：

```javascript
let allExtensions = Codec.getAllExtensions()
```

### Codec 事件

Codec 继承自 `EventSystem`，支持自定义事件：

```javascript
// 派发事件
this.dispatchEvent('compile', { content: '' })

// 监听事件
codec.on('parsed', function() { ... })
```

常用事件包括：
- `'compile'` — 编译时触发，携带 `{ content: '' }`
- `'parsed'` — 解析完成时触发

### 完整示例

这是一个完整的 codec 注册示例，展示了如何创建自定义文件格式的导入/导出：

```javascript
// 注册格式
new ModelFormat('my_custom', {
    icon: 'extension',
    name: '自定义格式',
    description: '一种自定义模型格式',
    bone_rig: true,
    rotate_cubes: true,
    box_uv: false,
})

// 注册编解码器
new Codec('my_custom', {
    name: '自定义格式',
    extension: 'custom',
    remember: true,
    format: Formats.my_custom,
    load_filter: {
        type: 'json',
        extensions: ['custom', 'custom_json']
    },
    load: function(model, file, args) {
        Formats.my_custom.select()
        this.parse(model, file.path)
    },
    parse: function(data, path, args) {
        // 将 JSON 数据解析到项目中
        new Cube({ from: [0, 0, 0], to: [16, 16, 16] }).addTo()
        Blockbench.dispatchEvent('load_project', { model: data, path })
    },
    compile: function(options) {
        // 将项目导出为 JSON
        return JSON.stringify({
            format: 'custom',
            elements: Cube.all.map(c => ({
                from: c.from,
                to: c.to
            }))
        })
    },
    export_options: {
        pretty_print: {
            label: '美化输出',
            type: 'checkbox',
            value: true
        }
    }
})
```

## 4. 全局 I/O 函数

### loadModelFile(file, args)

加载模型文件到 Blockbench。根据文件扩展名匹配对应的 Codec 并调用其 `load` 方法。

**处理流程：**
1. 检查是否存在已打开的同路径标签页
2. 获取文件扩展名
3. 依次尝试匹配 `image`、`text`、`json` 类型的 codec
4. 若匹配成功，调用 `codec.load(content, file, args)` 并返回
5. 若无匹配，调用 `unsupportedFileFormatMessage()`

```javascript
loadModelFile({ path: 'model.bbmodel', content: jsonData })
loadModelFile(file, { import_to_current_project: true })
```

### loadImages(files, event)

加载图片文件。弹出选择对话框让用户选择处理方式：

- `replace_texture` — 替换当前纹理
- `texture` — 添加为新纹理
- `layer` — 添加为纹理图层
- `reference_image` — 添加为参考图片
- `edit` — 在图片编辑器中打开
- `minecraft_skin` — 作为 Minecraft 皮肤打开
- `extrude_with_cubes` — 使用挤出工具

```javascript
let files = [{ path: 'tex.png', content: 'data:image/png;...' }]
await loadImages(files, event)
```

### unsupportedFileFormatMessage(file_path)

显示不支持的文件格式消息框。会检查是否有插件声明支持该扩展名，并提供安装建议。

```javascript
unsupportedFileFormatMessage('model.unknown')
```

## 5. 内部 I/O 辅助工具

### compileJSON(data, options?)

将对象编译为格式化或压缩的 JSON 字符串：

```javascript
let json = compileJSON(model, { small: true })   // 压缩模式
let json = compileJSON(model, { small: false })  // 美化模式
```

### autoParseJSON(content, options?)

自动解析 JSON 内容。支持处理 BOM 标记、注释和 trailing commas：

```javascript
let data = autoParseJSON(fileContent, { file_path: 'model.json' })
```
