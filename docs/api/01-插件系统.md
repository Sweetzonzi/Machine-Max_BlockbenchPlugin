# Blockbench 插件系统 API 文档

## 概述

Blockbench 的插件系统允许开发者扩展软件功能。插件可以通过商店安装、从本地文件加载或从 URL 加载。本文档涵盖插件的注册、配置、生命周期管理及核心 API。

---

## 1. 插件的注册和配置

### Plugin.register(id, data) - 静态方法

插件的官方注册入口。每个插件文件必须在执行时调用此方法向系统注册自身。

**参数说明：**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | `string` | 插件唯一 ID，须与 `bbplugin.json` 或商店中的 ID 一致 |
| data | `PluginOptions` | 插件配置对象 |

**处理流程：**

1. 检查 `Plugins.registered` 中是否已存在同名插件
2. 若不存在且存在 `unknown` 占位插件，则替换其 ID 并接管
3. 若不存在且无占位插件，显示加载失败提示
4. 若已存在，则调用 `plugin.extend(data)` 合并配置
5. 如果插件可安装（`isInstallable() === true`）且未禁用且定义了 `onload`，则自动调用 `runOnLoad()`

**返回值：** `Plugin | undefined` — 返回注册的插件实例，失败时返回 `undefined`。

### PluginOptions 接口

插件注册时使用的配置对象，定义插件的基本信息、生命周期回调及功能声明。

```typescript
interface PluginOptions {
    title: string                    // 插件标题
    author: string                   // 作者
    description?: string             // 插件描述（显示在插件浏览器中）
    about?: string                   // 详细的关于文本（Markdown 格式）
    version?: string                 // 版本号，如 "1.0.0"
    icon: string                     // 图标名称（FontAwesome，如 "fa-cube"）或图片文件名（如 "icon.png"）
    tags?: [string, string?, string?] // 标签（最多 3 个）
    variant: 'desktop' | 'web' | 'both' // 平台适配类型
    min_version?: string             // 最小 Blockbench 版本
    max_version?: string             // 最大 Blockbench 版本
    await_loading?: boolean          // 是否必须在项目打开前完成加载（如添加格式的插件）
    new_repository_format?: boolean  // 是否使用新的仓库格式（插件、图标、about 存储在独立文件夹中）
    contributes?: {                  // 声明插件贡献的功能
        formats?: string[]           // 添加的格式 ID 列表
        open_extensions?: string[]   // 添加的可打开文件扩展名列表
    }
    creation_date?: string           // 创建日期
    has_changelog?: boolean          // 是否有更新日志
    deprecation_note?: string        // 弃用说明（配合 "Deprecated" 标签使用）
    website?: string                 // 插件网站链接
    repository?: string              // 源代码仓库链接
    bug_tracker?: string             // 问题追踪链接
    contributors?: string[]          // 贡献者列表
    dependencies?: string[]          // 依赖的其他插件 ID 列表
    disabled?: boolean               // 是否禁用

    // 生命周期回调
    onload?(): void                  // 插件加载完成时调用
    onunload?(): void                // 插件卸载时调用
    oninstall?(): void               // 插件首次安装时调用
    onuninstall?(): void             // 插件被卸载时调用
}
```

#### 字段详细说明

**基本信息字段：**

- `title`（必需）— 插件标题，显示在插件浏览器和管理器中
- `author`（必需）— 作者名称
- `description` — 简短描述，显示在插件浏览器的列表视图中
- `about` — 详细的介绍文本，支持 Markdown 格式，当用户在插件浏览器中展开插件时显示
- `version` — 语义化版本号，默认为 `"0.0.1"`
- `icon`（必需）— 图标，可以使用 FontAwesome 图标名称（如 `"fa-cube"`）或图片文件名（如 `"icon.png"`、`"icon.svg"`）
- `tags` — 最多 3 个标签，用于在商店中分类和搜索

**平台与兼容性：**

- `variant`（必需）— 指定插件运行的平台：
  - `'desktop'` — 仅桌面版（Electron 应用）
  - `'web'` — 仅网页版（浏览器/PWA）
  - `'both'` — 所有平台
- `min_version` — 最低 Blockbench 版本要求，低于此版本将无法安装
- `max_version` — 最高 Blockbench 版本限制，高于此版本将无法安装

**功能声明：**

- `await_loading` — 设置为 `true` 时，插件必须在项目打开之前完成加载。适用于需要注册新格式的插件
- `contributes` — 声明插件提供的功能，即使插件未安装也能让 Blockbench 感知其存在：
  - `formats` — 插件注册的模型格式 ID 列表
  - `open_extensions` — 插件支持打开的文件扩展名列表

### 代码示例

```javascript
Plugin.register('my_plugin', {
    title: '我的插件',
    author: '作者名',
    description: '这是一个示例插件',
    about: '## 关于\n\n这是一个用 Markdown 编写的详细说明。',
    version: '1.0.0',
    icon: 'fa-cube',
    tags: ['工具', '实用'],
    variant: 'both',
    min_version: '4.0.0',
    dependencies: ['another_plugin'],
    contributes: {
        formats: ['my_custom_format']
    },
    onload() {
        Blockbench.showQuickMessage('我的插件已加载！');
        // 注册菜单、动作、面板等
    },
    onunload() {
        // 清理资源、移除事件监听等
        console.log('插件已卸载');
    },
    oninstall() {
        // 首次安装时的初始化逻辑
    },
    onuninstall() {
        // 卸载时的清理工作
    }
});
```

## 2. Plugin 类

`Plugin` 类代表一个插件实例，包含插件的元数据、状态及生命周期管理方法。

### 2.1 构造函数

```typescript
constructor(id: string = 'unknown', data?: PluginOptions | PluginSetupOptions)
```

**参数说明：**

| 参数 | 类型 | 说明 |
|------|------|------|
| id | `string` | 插件 ID，默认为 `'unknown'` |
| data | `PluginOptions \| PluginSetupOptions` | 可选的配置数据 |

构造函数初始化所有属性为默认值，然后调用 `extend(data)` 合并传入的配置，最后将自身添加到 `Plugins.all` 数组中。

### 2.2 核心属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 插件唯一 ID |
| `title` | `string` | 插件标题 |
| `author` | `string` | 作者名称 |
| `description` | `string` | 简短描述 |
| `about` | `string` | 详细介绍（Markdown） |
| `icon` | `string` | 图标名称或图片文件名 |
| `tags` | `string[]` | 标签列表 |
| `dependencies` | `string[]` | 依赖的其他插件 ID 列表 |
| `contributors` | `string[]` | 贡献者列表 |
| `version` | `string` | 版本号 |
| `variant` | `'desktop' \| 'web' \| 'both'` | 平台适配类型 |
| `min_version` | `string` | 最小 Blockbench 版本 |
| `max_version` | `string` | 最大 Blockbench 版本 |
| `deprecation_note` | `string` | 弃用说明 |
| `path` | `string` | 插件文件路径 |
| `website` | `string` | 网站链接 |
| `repository` | `string` | 仓库链接 |
| `bug_tracker` | `string` | 问题追踪链接 |
| `source` | `'store' \| 'file' \| 'url'` | 插件来源类型 |
| `creation_date` | `string \| number` | 创建日期 |
| `contributes` | `{ formats?: string[], open_extensions?: string[] }` | 贡献的功能声明 |
| `await_loading` | `boolean` | 是否需在项目打开前完成加载 |
| `has_changelog` | `boolean` | 是否有更新日志 |
| `changelog` | `PluginChangelog \| null` | 更新日志数据 |
| `about_fetched` | `boolean` | 是否已获取 about 内容 |
| `changelog_fetched` | `boolean` | 是否已获取更新日志 |
| `disabled` | `boolean` | 是否禁用 |
| `new_repository_format` | `boolean` | 是否使用新仓库格式 |
| `cache_version` | `number` | 缓存版本号（重载时递增） |
| `menu` | `Menu` | 插件的右键菜单实例 |
| `details` | `PluginDetails \| null` | 插件详细信息 |
| `uuid` | `UUID` | 插件唯一标识符（GUID） |
| `installed` | `boolean` | 是否已安装 |

#### 生命周期回调属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `onload` | `(() => void) \| undefined` | 插件加载完成时回调 |
| `onunload` | `(() => void) \| undefined` | 插件卸载时回调 |
| `oninstall` | `(() => void) \| undefined` | 插件首次安装时回调 |
| `onuninstall` | `(() => void) \| undefined` | 插件被卸载时回调 |

### 2.3 方法

#### extend(data)

将传入的数据对象的属性合并到当前插件实例中。

```typescript
extend(data: PluginOptions | PluginSetupOptions): this
```

**参数说明：**

| 参数 | 类型 | 说明 |
|------|------|------|
| data | `PluginOptions \| PluginSetupOptions` | 要合并的配置数据 |

**说明：** 使用 `Merge` 工具类进行类型安全的合并。字符串和布尔值属性会被直接覆盖，数组属性（tags、contributors、dependencies）会追加到现有数组末尾（tags 最多保留 3 个）。

**返回值：** `this` — 返回当前实例以支持链式调用。

---

#### install()

安装插件。

```typescript
async install(): Promise<void>
```

**处理流程：**

1. 如果插件带有 "Deprecated" 标签或 `deprecation_note`，显示弃用警告对话框，用户确认后继续
2. 检查会话安装次数，超过 12 次时显示提醒
3. 调用 `download(true)` 执行实际下载和安装

**返回值：** `Promise<void>`

---

#### load(first?, cb?)

加载插件。

```typescript
async load(first: boolean = false, cb?: (this: Plugin) => void): Promise<void>
```

**参数说明：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| first | `boolean` | `false` | 是否首次安装，若为 `true` 则触发 `oninstall` 回调 |
| cb | `((this: Plugin) => void) \| undefined` | — | 加载完成后的回调函数 |

**处理流程：**

1. 将自身注册到 `Plugins.registered[this.id]`
2. 根据来源类型和平台确定插件文件路径
3. 调用 `#runPluginFile(path)` 执行插件代码
4. 如果 `first` 为 `true`，触发 `oninstall()` 和 `installed_plugin` 事件
5. 标记 `installed = true`

**返回值：** `Promise<void>`

---

#### unload()

卸载插件（不删除注册信息）。

```typescript
unload(): this
```

**处理流程：**

1. 调用 `onunload()` 回调
2. 派发 `unloaded_plugin` 事件

**返回值：** `this`

---

#### uninstall()

完全卸载插件。

```typescript
uninstall(): this
```

**处理流程：**

1. 调用 `unload()` 卸载
2. 调用 `onuninstall()` 回调
3. 派发 `uninstalled_plugin` 事件
4. 从 `Plugins.registered` 中移除
5. 从 `Plugins.installed` 中移除
6. 设置 `installed = false`、`disabled = false`
7. 若非商店来源且为桌面端，从 `Plugins.all` 中移除
8. 删除本地缓存的文件（`.js`、图标、`.about.md`）

**返回值：** `this`

---

#### reload()

重新加载插件（仅对 `file` 和 `url` 来源有效）。

```typescript
reload(): this
```

**处理流程：**

1. 递增 `cache_version`
2. 调用 `unload()` 卸载当前版本
3. 清空 tags、contributors、dependencies
4. 从 `Plugins.all` 中移除
5. 重置 details 和 changelog 状态
6. 根据 source 类型调用 `loadFromFile()` 或 `loadFromURL()` 重新加载
7. 刷新 about 和 changelog

**返回值：** `this`

---

#### runOnLoad()

执行 `onload` 回调。

```typescript
runOnLoad(): void
```

**处理流程：**

1. 设置 `Plugins.currently_loading` 为当前插件 ID
2. 调用 `this.onload()`
3. 重置 `Plugins.currently_loading` 为空
4. 派发 `loaded_plugin` 事件

---

#### download(first?)

下载插件文件（商店来源）。

```typescript
async download(first: boolean = false): Promise<void>
```

**参数说明：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| first | `boolean` | `false` | 是否首次安装 |

**处理流程：**

1. 调用 `installDependencies(first)` 处理依赖
2. 在桌面端，将插件文件（和图标、about）下载到本地插件目录
3. 在网页端，直接调用 `load(first)`
4. 调用 `load(first)` 加载插件

**返回值：** `Promise<void>`

---

#### loadFromFile(file, first?)

从本地文件加载插件。

```typescript
async loadFromFile(file: Filesystem.FileResult, first: boolean = false): Promise<this | void>
```

**参数说明：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| file | `Filesystem.FileResult` | — | 文件对象，包含 `path`、`name`、`content` |
| first | `boolean` | `false` | 是否首次安装 |

**处理流程：**

1. 首次加载时显示确认对话框
2. 从文件路径提取插件 ID
3. 注册到 `Plugins.registered` 和 `Plugins.all`
4. 设置 `source = 'file'`，添加 "Local" 标签
5. 在桌面端调用 `#runPluginFile()` 执行代码；在网页端调用 `#runCode()` 执行
6. 首次安装时触发 `oninstall()`
7. 标记 `installed = true`，保存状态

**返回值：** `Promise<this | void>`

---

#### loadFromURL(url, first?)

从 URL 加载插件。

```typescript
async loadFromURL(url: string, first: boolean = false): Promise<this | void>
```

**参数说明：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| url | `string` | — | 插件文件的 URL |
| first | `boolean` | `false` | 是否首次安装 |

**处理流程：**

1. 首次加载时显示确认对话框
2. 从 URL 提取插件 ID
3. 注册到 `Plugins.registered` 和 `Plugins.all`
4. 添加 "Remote" 标签，设置 `source = 'url'`
5. 通过 AJAX 获取并执行插件代码
6. 若获取失败且为桌面端，尝试从本地缓存加载
7. 在桌面端，将插件文件保存到本地缓存

**返回值：** `Promise<this | void>`

---

#### toggleDisabled()

切换插件的禁用/启用状态。

```typescript
toggleDisabled(): void
```

**处理流程：**

- 禁用时：设置 `disabled = true`，调用 `unload()`
- 启用时：调用 `runOnLoad()`，设置 `disabled = false`
- 保存状态到本地存储

---

#### isReloadable()

判断插件是否可重载。

```typescript
isReloadable(): boolean
```

**返回值：** `boolean` — 当插件已安装、未禁用、且为桌面端的 `file` 来源或 `url` 来源时返回 `true`。

---

#### isInstallable()

判断插件是否可在当前环境中安装。

```typescript
isInstallable(): true | string
```

**返回值：**

- `true` — 可安装
- `string` — 不可安装的原因描述，可能的值包括：
  - `"仅桌面端"`（`app_only`）— 插件仅支持桌面端
  - `"仅网页端"`（`web_only`）— 插件仅支持网页端
  - `"Blockbench 版本过低"`（`outdated_client`）— 当前版本低于 `min_version`
  - `"插件版本过低"`（`outdated_plugin`）— 当前版本高于 `max_version`

## 3. Plugins 全局对象

`Plugins` 是全局的插件管理对象，管理所有插件的注册、加载和状态。

### 3.1 属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `dialog` | `Dialog \| null` | 插件管理窗口的 Dialog 实例 |
| `Vue` | `Vue \| null` | 插件管理窗口的 Vue 实例 |
| `installed` | `PluginInstallation[]` | 已安装插件的配置列表 |
| `json` | `any` | 商店插件 JSON 数据（从 API 获取的插件清单） |
| `download_stats` | `Record<string, number>` | 插件下载统计数据 |
| `all` | `Plugin[]` | 所有插件实例（包括已安装和商店中未安装的） |
| `registered` | `Record<string, Plugin>` | 按 ID 注册的插件映射表 |
| `currently_loading` | `string` | 当前正在加载的插件 ID |
| `loading_promise` | `Promise \| null` | 商店插件清单加载的 Promise |
| `api_path` | `string` | 插件 API 的基础路径 |
| `path` | `string` | 本地插件文件存储路径 |

### 3.2 方法

#### devReload()

开发模式重载所有 `file` 来源的插件。

```typescript
devReload(): void
```

**说明：** 遍历 `Plugins.all`，对所有 `source === 'file'` 且可重载的插件调用 `reload()`。完成后显示重载数量提示。

#### sort()

按下载量对所有插件排序。

```typescript
sort(): void
```

**说明：** 排序规则：弃用插件排在最后，其余按下载量降序排列，下载量相同时按标题字母顺序排列。

## 5. 插件加载流程

插件系统的完整加载流程如下：

### 5.1 启动阶段

```
应用启动
    │
    ▼
boot_loader.js 加载所有核心模块
    │
    ▼
main.ts 调用 loadInstalledPlugins()
```

### 5.2 loadInstalledPlugins() 详细流程

```
loadInstalledPlugins()
    │
    ├── 等待 Plugins.loading_promise 完成（商店 JSON 清单加载）
    │
    ├── 从商店 JSON 创建 Plugin 实例（所有可用插件）
    │   └── 对每个插件 ID：new Plugin(id, Plugins.json[id])
    │
    ├── 解析依赖顺序
    │   └── 递归调整 Plugins.installed 数组顺序，使依赖项排在前面
    │
    └── 按顺序加载已安装的插件
        │
        ├── file 来源（开发插件）
        │   └── 创建实例 → loadFromFile() → 执行代码
        │
        ├── url 来源（远程插件）
        │   └── 创建实例 → loadFromURL() → 下载并执行代码
        │
        └── store 来源（商店插件）
            ├── 已缓存且版本匹配 → load() → 从本地文件执行
            └── 需更新 → download() → 下载最新版 → load()
```

### 5.3 插件内部注册流程

```
插件代码执行时调用 Plugin.register(id, data)
    │
    ├── 查找 Plugins.registered[id]
    │   ├── 存在 → extend(data) 合并配置
    │   └── 不存在 → 查找 unknown 占位 → 替换 ID
    │
    ├── 检查 isInstallable() && !disabled && onload 存在
    │
    └── 调用 runOnLoad()
        ├── 设置 currently_loading
        ├── 执行 onload()
        ├── 重置 currently_loading
        └── 派发 loaded_plugin 事件
```

## 6. 插件权限系统

插件在桌面端（Electron）通过受限的 `requireNativeModule` API 获取 Node.js 功能访问权限。

### 权限机制

- **作用域限制：** `fs` 模块通过 `scope` 参数限定文件访问范围，插件只能读写指定目录内的文件
- **按需授予：** 插件在清单中声明所需权限，用户在安装时确认
- **动态撤销：** 用户可在插件管理器中随时撤销已授予的权限，无需重启即可生效

### 相关方法

| 方法 | 说明 |
|------|------|
| `getPluginPermissions(plugin)` | 获取插件的当前权限列表 |
| `revokePluginPermissions(plugin)` | 撤销插件的所有权限，返回被撤销的权限列表 |
| `getPluginScopedRequire(plugin)` | 获取插件的带作用域限制的 require 函数 |

### 权限管理界面

在插件上下文菜单中，"撤销权限" 菜单项（`dialog.plugins.revoke_permissions`）在插件拥有权限时可用。点击后撤销所有权限并提示重启生效。

### 权限显示

在 `getPluginDetails()` 返回的详细信息中，`permissions` 字段会列出插件拥有的所有权限。对于 `fs` 模块，会显示其允许访问的目录路径。

## 7. 全局变量与别名

为了兼容性和易用性，Blockbench 将插件相关 API 暴露为全局变量：

```javascript
// Plugin 类的别名
const BBPlugin = Plugin;
typeof BBPlugin === typeof Plugin; // true

// 全局可用
window.Plugins = Plugins;
window.Plugin = Plugin;
window.BBPlugin = Plugin;
```

这意味着在插件代码中可以直接使用 `Plugin`、`BBPlugin` 和 `Plugins` 而无需导入。

## 8. 完整示例

以下是一个完整的插件示例，演示了注册、生命周期管理和基本功能：

```javascript
Plugin.register('example_plugin', {
    title: '示例插件',
    author: 'Blockbench 开发者',
    description: '演示插件系统 API 的用法',
    about: '## 示例插件\n\n这是一个演示插件，展示了 Blockbench 插件系统的核心 API。',
    version: '1.0.0',
    icon: 'fa-star',
    tags: ['示例', '教程'],
    variant: 'both',
    min_version: '4.0.0',

    onload() {
        console.log('示例插件已加载');

        // 注册一个新动作
        new Action('example_action', {
            name: '示例动作',
            description: '点击执行示例功能',
            icon: 'fa-star',
            category: 'plugins',
            click() {
                Blockbench.showQuickMessage('示例插件正在运行！');
            }
        });

        // 添加菜单项
        Plugin.menu.add({
            name: '示例菜单项',
            icon: 'star',
            condition: plugin => plugin.id === 'example_plugin',
            click() {
                Blockbench.showQuickMessage('菜单项被点击');
            }
        });
    },

    onunload() {
        // 清理动作和菜单
        Action.actions.example_action?.delete();
        console.log('示例插件已卸载');
    },

    oninstall() {
        console.log('示例插件已安装');
        Blockbench.showQuickMessage('感谢安装示例插件！');
    },

    onuninstall() {
        console.log('示例插件已卸载');
        Blockbench.showQuickMessage('示例插件已被卸载');
    }
});
```
