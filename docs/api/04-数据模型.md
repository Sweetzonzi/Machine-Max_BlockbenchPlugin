## 1. 向量类型别名

Blockbench 中使用元组（Tuple）类型表示多维向量，作为三维空间操作的基础数据类型。

```javascript
/** 四维向量：[x, y, z, w] */
ArrayVector4 = [number, number, number, number]

/** 三维向量：[x, y, z] */
ArrayVector3 = [number, number, number]

/** 二维向量：[u, v] */
ArrayVector2 = [number, number]
```

### 使用示例

```javascript
// 定义立方体的起始与结束坐标
const from = [-8, 0, -8];   // ArrayVector3
const to   = [8, 16, 8];    // ArrayVector3

// UV 偏移
const uvOffset = [0, 0];    // ArrayVector2
```

## 2. ModelProject 类 — 模型项目

`ModelProject` 是 Blockbench 中最顶层的模型容器。一个项目对应一个打开的 `.bbmodel` 文件，标签栏可用于在多个项目间切换。

### 构造函数

```javascript
new ModelProject(options?, uuid?)
```

- **options**：可选，初始配置对象
- **uuid**：可选，指定 UUID，未传则自动生成

### 关键属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `uuid` | `UUID` | 项目唯一标识符 |
| `selected` | `boolean` | 是否被选中（当前激活） |
| `locked` | `boolean` | 是否锁定，锁定后不可编辑 |
| `box_uv` | `boolean` | 是否使用 Box UV 模式 |
| `model_identifier` | `string` | 模型标识符，用于导出标识 |
| `save_path` | `string` | 文件保存路径 |
| `export_path` | `string` | 导出路径 |
| `format` | `ModelFormat` | 当前格式（如 `'java_block'`、`'bedrock'` 等） |
| `mode` | `string` | 当前模式 ID（如 `'edit'`、`'paint'`） |
| `view_mode` | `string` | 视图模式（如 `'textured'`、`'solid'`、`'wireframe'`） |
| `elements` | `OutlinerElement[]` | 所有元素列表（Cube、Mesh 等） |
| `groups` | `Group[]` | 所有组/骨骼列表 |
| `selected_elements` | `OutlinerElement[]` | 当前选中的元素 |
| `textures` | `Texture[]` | 所有纹理定义 |
| `outliner` | `OutlinerNode[]` | 大纲树的根节点数组 |
| `animations` | `_Animation[]` | 动画列表 |
| `display_settings` | `object` | 显示槽位设置（如 `'head'`、`'right_arm'` 等） |

### 关键方法

#### 项目生命周期

**`project.select()`**
选中当前项目，将其标签页切换为激活状态。

```javascript
// 切换到第一个项目
ModelProject.all[0].select();
```

**`project.unselect(closing?)`**
取消选中项目。`closing` 参数表示是否正在关闭项目。

**`project.close(force?)`**
关闭项目。如果存在未保存的更改且 `force` 不为 `true`，则弹出保存确认对话框。

```javascript
// 强制关闭项目，不提示保存
project.close(true);
```

#### 项目属性操作

**`project.extend(object)`**
用给定对象扩展项目属性，相当于 `Object.assign`。

```javascript
project.extend({
    box_uv: true,
    model_identifier: 'my_model'
});
```

#### 查询方法

**`project.getDisplayName(with_extension?)`**
获取项目显示名称。`with_extension` 为 `true` 时包含文件扩展名。

```javascript
const name = project.getDisplayName(true);
// 例如: "my_model.bbmodel"
```

**`project.getFileExtension()`**
获取模型文件的扩展名字符串。

```javascript
const ext = project.getFileExtension();
// 例如: ".bbmodel"
```

**`project.getUVWidth(texture?)`**
获取指定纹理的 UV 宽度。未传纹理时返回当前纹理或默认值。

**`project.getUVHeight(texture?)`**
获取指定纹理的 UV 高度。未传纹理时返回当前纹理或默认值。

```javascript
const w = project.getUVWidth();   // 纹理像素宽度
const h = project.getUVHeight();  // 纹理像素高度
```

#### UI 交互

**`project.showContextMenu(event)`**
在鼠标事件位置显示项目右键菜单。

**`project.updateThumbnail()`**
更新项目的缩略图预览。

### 静态属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `ModelProject.all` | `ModelProject[]` | 所有已打开的项目实例数组 |

```javascript
// 遍历所有项目
ModelProject.all.forEach(proj => {
    console.log(`项目: ${proj.getDisplayName()}`);
});
```

## 3. OutlinerNode 类 — 大纲节点基类

`OutlinerNode` 是所有大纲节点的抽象基类。`Group`（组/骨骼）和 `OutlinerElement`（元素）都继承自此类。大纲树（Outliner）中的每一个条目都是一个 `OutlinerNode`。

### 属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `name` | `string` | 节点名称，显示在大纲中的文本 |
| `uuid` | `UUID` | 全局唯一标识符，一旦创建不可变更 |
| `export` | `boolean` | 是否导出该节点 |
| `locked` | `boolean` | 是否锁定，锁定后不可编辑 |
| `scope` | `number` | 作用域层级 |
| `parent` | `OutlinerNode & ParentTraits \| 'root'` | 父节点，根节点时为 `'root'` |
| `selected` | `boolean` | 是否被选中 |
| `children?` | `OutlinerNode[]` | 子节点数组（仅容器节点如 Group 有此属性） |
| `type` | `string` | 节点类型标识（如 `'cube'`、`'group'`、`'mesh'`） |

### 方法

#### 生命周期

**`node.init()`**
初始化节点并加入项目的大纲树。此方法会将节点注册到 `OutlinerNode.all` 列表中。

```javascript
const group = new Group({ name: '新骨骼' });
group.init(); // 加入大纲
```

**`node.addTo(target?, index?)`**
将节点添加到指定的父节点下，可选指定插入索引位置。

```javascript
const parentGroup = new Group({ name: '父组' }).init();
const childGroup = new Group({ name: '子组' }).init();
childGroup.addTo(parentGroup); // 将子组添加到父组下
```

**`node.removeFromParent()`**
将节点从其父节点移除，但不销毁节点本身。

#### 选择与显示

**`node.select(event?, outliner_click?)`**
选中节点。当 `outliner_click` 为 `true` 时，按大纲点击行为处理（触发 Shift 多选等）。

**`node.unselect()`**
取消选中节点。

**`node.showInOutliner()`**
展开大纲树并滚动到该节点位置，使其在视图中可见。

```javascript
// 选中并聚焦到某个骨骼
someGroup.select();
someGroup.showInOutliner();
```

#### 层级关系

**`node.getAllAncestors()`**
获取节点的所有祖先节点数组（从直接父节点到根节点）。

```javascript
const ancestors = node.getAllAncestors();
ancestors.forEach(anc => console.log(anc.name));
```

**`node.sortInBefore(element?)`**
将当前节点插入到指定 `element` 之前。不传参数时等同于移到同级末尾。

**`node.isChildOf(node, max_levels?)`**
检查当前节点是否为指定 `node` 的子节点。`max_levels` 限制最大检查层级。

```javascript
if (child.isChildOf(parent)) {
    console.log('child 是 parent 的后代');
}
```

**`node.getDepth()`**
获取节点在大纲树中的深度（根节点深度为 0）。

#### 命名

**`node.rename()`**
进入重命名模式（弹出内联编辑框）。

**`node.saveName(save?)`**
保存当前编辑的名称。`save` 参数控制是否持久化。

**`node.sanitizeName()`**
清理名称中的非法字符，返回合法名称字符串。

**`node.createUniqueName(additional?)`**
基于当前名称创建唯一名称，避免与同级节点重名。`additional` 为可选后缀。

```javascript
const uniqueName = node.createUniqueName('_copy');
// 例如: "骨骼_copy" 或 "骨骼_copy_2"
```

#### 搜索与过滤

**`node.matchesFilter(search_term)`**
检查节点名称是否匹配搜索词，用于大纲搜索过滤。

```javascript
const results = OutlinerNode.all.filter(n => n.matchesFilter('door'));
```

#### 编辑操作

**`node.remove()`**
从项目中移除节点。会从 `OutlinerNode.all` 和父节点中同时移除。

**`node.duplicate()`**
复制节点，返回新的节点实例。`Group` 会深拷贝子节点。

```javascript
const copy = original.duplicate();
copy.addTo(original.parent); // 将复制体添加到同级
```

**`node.showContextMenu(event)`**
在鼠标事件位置显示节点右键菜单。

#### 视图更新

**`node.updateElement()`**
更新关联的 Vue 组件节点，刷新大纲显示。

### 静态属性与方法

| 静态成员 | 类型 / 签名 | 说明 |
|----------|-------------|------|
| `OutlinerNode.all` | `OutlinerNode[]` | 所有大纲节点 |
| `OutlinerNode.selected` | `OutlinerNode[]` | 所有选中节点 |
| `OutlinerNode.uuids` | `{ [uuid]: OutlinerNode }` | UUID 到节点的映射字典 |

**`OutlinerNode.addBehaviorOverride(options)`**
添加行为覆盖（Behavior Override），用于自定义节点的行为逻辑。

```javascript
OutlinerNode.addBehaviorOverride({
    type: 'group',
    condition: () => Project.format.id === 'custom_format',
    actions: {
        canDelete: () => false
    }
});
```

## 4. OutlinerElement 类 — 大纲元素基类

继承自 `OutlinerNode`，代表具体模型元素（如 `Cube`、`Mesh` 等）。所有可渲染的几何体都继承自此类的接口。

### 附加功能

#### 初始化与注册

**`element.init()`**
初始化元素并注册到 `Project.elements` 列表中。与 `OutlinerNode.init()` 不同，此方法额外将元素加入 `OutlinerElement` 的类级别跟踪。

```javascript
const cube = new Cube({ name: '我的立方体', from: [-4,0,-4], to: [4,8,4] });
cube.init(); // 加入 Project.elements
```

#### 选择系统（增强）

**`element.select(event?, is_outliner_click?)`**
选中元素。当 `is_outliner_click` 为 `true` 时，支持以下交互：
- **Shift 点击**：范围多选，选中从上一次选中项到当前项之间的所有元素
- **Ctrl 点击**：切换选中状态，将当前元素加入或移出已选集合

```javascript
// 程序化选中
element.select(null, false);
```

**`element.clickSelect(event)`**
鼠标点击选择入口，由事件系统自动调用。

**`element.unselect()`**
取消选中该元素。

**`element.markAsSelected()`**
仅标记选中状态，不触发选择事件。

#### 批量操作

**`element.forSelected(fc, undo_tag?)`**
对所有处于选中状态且类型相同的元素执行回调函数。`undo_tag` 可选，传入后将操作包裹在单个撤销动作内。

```javascript
// 将所有选中的立方体名称加上 "_batch" 后缀
cube.forSelected(function(el) {
    el.name += '_batch';
}, '批量重命名');
```

#### 复制与序列化

**`element.duplicate()`**
复制元素，返回新的 `OutlinerElement` 实例。

**`element.getUndoCopy()`**
获取用于撤销的浅拷贝数据，记录元素当前状态的快照。

**`element.getSaveCopy()`**
获取用于持久化保存的数据副本，过滤掉运行时状态。

### 静态方法

| 静态方法 | 签名 | 说明 |
|----------|------|------|
| `OutlinerElement.fromSave(obj, keep_uuid?)` | `(obj, boolean?) => OutlinerElement` | 从保存的 JSON 数据恢复元素实例 |
| `OutlinerElement.isTypePermitted(type)` | `(string) => boolean` | 检查元素类型是否被当前格式允许 |
| `OutlinerElement.hasAny()` | `() => boolean` | 当前项目是否包含任何该类型元素 |
| `OutlinerElement.hasSelected()` | `() => boolean` | 是否有选中元素 |
| `OutlinerElement.registerType(constructor, id)` | `(constructor, string) => void` | 注册新的元素类型，扩展 Blockbench 支持的几何类型 |

```javascript
// 从 JSON 恢复
const saved = {
    name: 'cube',
    from: [-8, 0, -8],
    to: [8, 16, 8],
    type: 'cube'
};
const restored = OutlinerElement.fromSave(saved);
restored.init();

// 注册自定义元素类型
class MyElement extends OutlinerElement {
    // 自定义实现
}
OutlinerElement.registerType(MyElement, 'my_element');
```

### 静态属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `OutlinerElement.types` | `Record<string, ElementTypeConstructor>` | 已注册的元素类型构造器字典 |

## 5. Group 类 — 组/骨骼

继承自 `OutlinerNode`，是容器节点，可包含子节点（其他 Group 或 OutlinerElement）。在骨骼动画模型中，Group 用作骨骼（Bone）。

### 构造函数

```javascript
new Group({
    name: '骨骼名称',
    origin: [0, 0, 0],
    rotation: [0, 0, 0]
});
```

```javascript
// 创建骨骼链：身体 → 左臂
const body = new Group({
    name: 'body',
    origin: [0, 24, 0]
}).init();

const leftArm = new Group({
    name: 'left_arm',
    origin: [-5, 22, 0],
    rotation: [0, 0, -10]
}).init();

leftArm.addTo(body);
```

### 属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `children` | `OutlinerNode[]` | 直接子节点列表 |
| `origin` | `ArrayVector3` | 骨骼原点坐标（旋转中心） |
| `rotation` | `ArrayVector3` | 旋转角度（欧拉角，单位：度） |
| `reset` | `boolean` | 是否在导入时重置变换 |
| `shade` | `boolean` | 是否着色渲染 |
| `visibility` | `boolean` | 可见性 |
| `autouv` | `0 \| 1 \| 2` | 自动 UV 模式（0=关闭，1=开启，2=智能） |
| `mirror_uv` | `boolean` | 是否 UV 镜像 |
| `isOpen` | `boolean` | 大纲中是否展开 |
| `ik_enabled` | `boolean` | 是否启用反向动力学（IK） |
| `ik_chain_length` | `number` | IK 链长度 |
| `texture?` | `string` | 关联纹理标识 |
| `bedrock_binding?` | `string` | 基岩版 Molang 绑定表达式 |
| `mesh` | `THREE.Object3D` | Three.js 三维对象引用，用于 3D 视图渲染 |

### 方法

**`group.extend(options)`**
用给定对象扩展组的属性，返回 `this` 支持链式调用。

```javascript
group.extend({
    visibility: true,
    shade: true
}).init();
```

**`group.selectChildren(event)`**
选中当前组的所有直接子节点，返回 `this`。

```javascript
group.selectChildren(); // 批量选中所有子级
```

**`group.openUp()`**
递归展开所有父节点，确保本组在大洋中可见，返回 `this`。

**`group.remove(undo?)`**
从项目中移除组及其所有子节点。`undo` 参数控制是否可撤销。

**`group.resolve()`**
移除组但保留所有子元素，将子元素提升到父级。返回提升后的节点数组。

```javascript
// 将 "wrapper" 组移除，其子节点直接挂到祖父节点下
wrapper.resolve();
```

**`group.transferOrigin(origin)`**
转移骨骼原点。将组及其子元素的原点整体偏移到新坐标，所有子元素的相对位置自动调整。

```javascript
group.transferOrigin([0, 12, 0]);
```

**`group.duplicate()`**
深度复制组及其所有子节点，返回新的 `Group` 实例。

**`group.sortContent()`**
对子元素按名称排序，返回 `this`。

**`group.forEachChild(callback, type?, for_self?)`**
遍历子节点执行回调。`type` 可按类型筛选；`for_self` 为 `true` 时对自身也执行回调。

```javascript
group.forEachChild(child => {
    child.export = true;
}, 'cube', false);
```

### 静态属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `Group.all` | `Group[]` | 所有组实例 |
| `Group.selected` | `Group[]` | 所有选中的组 |
| `Group.first_selected` | `Group \| undefined` | 第一个选中的组 |
| `Group.multi_selected` | `Group[]` | 多选状态下的组（与 `selected` 相同但语义更明确） |

## 6. Cube 类 — 立方体

继承自 `OutlinerElement`，是 Blockbench 中最基本的几何元素。由六个面（北、南、东、西、上、下）围合而成。

### 构造函数

```javascript
new Cube({
    name: '立方体',
    from: [-8, 0, -8],      // 起始坐标
    to: [8, 16, 8],          // 结束坐标
    origin: [0, 0, 0],       // 旋转原点
    rotation: [0, 0, 0],     // 旋转角度
    faces: {                  // 六个方向的面
        north: { uv: [0, 0, 16, 16] },
        south: { uv: [0, 0, 16, 16] },
        east:  { uv: [0, 0, 16, 16] },
        west:  { uv: [0, 0, 16, 16] },
        up:    { uv: [0, 0, 16, 16] },
        down:  { uv: [0, 0, 16, 16] }
    }
}).init();
```

### 简单示例

```javascript
const head = new Cube({
    name: 'head',
    from: [-4, 0, -4],
    to: [4, 8, 4],
    origin: [0, 24, 0],
    faces: {
        north: { uv: [0, 0, 8, 8] },
        south: { uv: [16, 0, 24, 8] },
        east:  { uv: [8, 0, 16, 8] },
        west:  { uv: [24, 0, 32, 8] },
        up:    { uv: [8, 0, 16, 8], rotation: 90 },
        down:  { uv: [16, 0, 24, 8], rotation: -90 }
    }
}).init();
```

### 属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `from` | `ArrayVector3` | 起始坐标（最小的 x、y、z） |
| `to` | `ArrayVector3` | 结束坐标（最大的 x、y、z） |
| `origin` | `ArrayVector3` | 旋转原点 |
| `rotation` | `ArrayVector3` | 欧拉旋转角度 |
| `stretch` | `ArrayVector3` | 拉伸值 |
| `autouv` | `0 \| 1 \| 2` | 自动 UV 模式（0=关闭，1=开启，2=智能） |
| `shade` | `boolean` | 是否着色 |
| `mirror_uv` | `boolean` | UV 是否镜像 |
| `box_uv` | `boolean` | 是否使用 Box UV（将纹理映射到立方体六个面上） |
| `inflate` | `number` | 膨胀值，使立方体沿法线方向扩展 |
| `visibility` | `boolean` | 是否可见 |
| `color` | `number` | 颜色值 |
| `uv_offset` | `ArrayVector2` | UV 坐标的整体偏移量 |
| `faces` | `Record<string, CubeFace>` | 六个方向的面配置（键名：`north`、`south`、`east`、`west`、`up`、`down`） |

### 方法

**`cube.size()`**
获取立方体的尺寸（宽、高、深），返回 `ArrayVector3`。

```javascript
const [w, h, d] = head.size();
console.log(`尺寸: ${w} x ${h} x ${d}`);
```

**`cube.rotationAxis()`**
获取立方体的旋转轴字符串。当只有一个轴有旋转时返回该轴名（`'x'`、`'y'` 或 `'z'`），多轴旋转时返回空字符串。

**`cube.getUndoCopy()`**
获取用于撤销的拷贝数据。

**`cube.getSaveCopy()`**
获取用于持久化保存的拷贝数据。

**`cube.roll(axis, steps, origin)`**
沿指定轴旋转立方体，每次旋转 90 度。`steps` 为正时顺时针，为负时逆时针。

```javascript
// 沿 Y 轴顺时针旋转 2 次（180 度）
cube.roll('y', 2, [0, 0, 0]);
```

**`cube.flip(axis, center)`**
沿指定轴翻转立方体。

```javascript
// 沿 X 轴翻转，以原点为中心
cube.flip('x', [0, 0, 0]);
```

**`cube.transferOrigin(origin)`**
将旋转原点转移到新位置，自动调整 `from` 和 `to` 使世界坐标不变。

**`cube.getWorldCenter()`**
获取立方体在世界坐标系中的中心点，返回 `THREE.Vector3`。

**`cube.setUVMode(box_uv)`**
设置 UV 模式。`box_uv = true` 时启用 Box UV。

**`cube.setColor(color)`**
设置立方体颜色。

**`cube.applyTexture(texture, faces)`**
将纹理应用到指定面上。`faces` 可指定应用的面列表，不传则应用到所有面。

```javascript
cube.applyTexture(project.textures[0], ['north', 'south']);
```

**`cube.mapAutoUV()`**
执行自动 UV 映射，根据立方体尺寸和纹理分辨率自动计算 UV 坐标。

**`cube.moveVector(offset, axis)`**
沿指定轴方向移动立方体。`axis` 为移动参考坐标系。

**`cube.resize(value, axis, negative)`**
沿指定轴调整立方体尺寸。`negative` 控制是调整 `from` 端还是 `to` 端。

```javascript
// 沿 Y 轴将立方体向上拉伸 4 个单位
cube.resize(4, 'y', false);
```

| 静态成员 | 说明 |
|----------|------|
| `Cube.all` | `Cube[]` — 所有 Cube 实例 |
| `Cube.selected` | `Cube[]` — 所有选中的 Cube 实例 |
| `Cube.hasAny()` | 是否有任何 Cube 元素 |
| `Cube.hasSelected()` | 是否有选中的 Cube 元素 |

## 7. Mesh 类 — 网格

继承自 `OutlinerElement`，表示由顶点、边和面构成的多边形网格。适用于复杂形状的建模。

### 构造函数

```javascript
new Mesh({
    name: '网格',
    vertices: {
        v0: [0, 0, 0],
        v1: [1, 0, 0],
        v2: [0, 1, 0]
    },
    faces: {
        f0: { vertices: ['v0', 'v1', 'v2'], uv: [[0,0], [1,0], [0,1]] }
    }
}).init();
```

### 属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `shading` | `'flat' \| 'smooth'` | 着色模式，`'flat'` 为平直着色，`'smooth'` 为平滑着色 |
| `vertices` | `Record<string, ArrayVector3>` | 顶点字典，键为顶点 ID（如 `'v0'`、`'v1'`） |
| `faces` | `Record<string, MeshFace>` | 面字典，键为面 ID（如 `'f0'`、`'f1'`） |
| `seams` | `Record<string, 'auto' \| 'divide' \| 'join'>` | UV 缝线映射，控制 UV 展开时的接缝行为 |
| `vertice_list` | `ArrayVector3[]` | 顶点列表（与 `vertices` 同步，但按索引访问） |

### 方法

**`mesh.getSelectedVertices()`**
获取选中的顶点 ID 数组。

```javascript
const selectedVerts = mesh.getSelectedVertices();
// 返回: ['v0', 'v3', 'v5']
```

**`mesh.getSelectedEdges()`**
获取选中的边数组，每条边由两个顶点 ID 组成的元组表示。

```javascript
const selectedEdges = mesh.getSelectedEdges();
// 返回: [['v0', 'v1'], ['v1', 'v2']]
```

**`mesh.getSelectedFaces()`**
获取选中的面 ID 数组。

**`mesh.calculateNormals()`**
计算并返回所有面的法线向量，返回 `Record<string, ArrayVector3>`。

**`mesh.addVertices(...vectors)`**
添加一个或多个顶点，返回新添加顶点 ID 数组。

```javascript
const newIds = mesh.addVertices([1, 1, 0], [0, 1, 1]);
// 返回: ['v3', 'v4']
```

**`mesh.addFaces(...faces)`**
添加一个或多个面，返回新添加面 ID 数组。

```javascript
mesh.addFaces({
    vertices: ['v0', 'v1', 'v3'],
    uv: [[0,0], [1,0], [0,1]]
});
```

**`mesh.applyTexture(texture, faces?)`**
将纹理应用到指定面。`faces` 可选，不传则应用到所有面。

**`mesh.flip(axis)`**
沿指定轴翻转网格的所有顶点坐标。

**`mesh.roll(axis, steps)`**
沿指定轴旋转网格，每次 90 度。

### 静态属性

| 属性 | 说明 |
|------|------|
| `Mesh.all` | `Mesh[]` — 所有 Mesh 实例 |
| `Mesh.selected` | `Mesh[]` — 所有选中的 Mesh 实例 |

## 8. SplineMesh 类 — 样条曲线网格

继承自 `OutlinerElement`，表示由样条曲线定义的网格，常用于管道、绳索等弧形物体。

### 属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `vertices` | `Record<string, ArrayVector3>` | 控制点顶点字典 |
| `handles` | `Record<string, SplineHandle>` | 贝塞尔手柄字典，控制曲线的切线方向 |
| `curves` | `Record<string, SplineCurve>` | 曲线段字典，定义顶点间的连接关系 |
| `cyclic` | `boolean` | 是否循环（首尾相连形成闭合环路） |
| `radial_resolution` | `number` | 径向分辨率，控制横截面的细分程度 |
| `tubular_resolution` | `number` | 管道分辨率，控制沿曲线的细分程度 |
| `render_mode` | `'mesh' \| 'path'` | 渲染模式，`'mesh'` 渲染为网格，`'path'` 渲染为路径线 |

```javascript
const spline = new SplineMesh({
    name: '管道',
    vertices: {
        p0: [0, 0, 0],
        p1: [10, 5, 0],
        p2: [20, 0, 0]
    },
    handles: {
        h0: { vertex: 'p0', next: [5, 5, 0] },
        h1: { vertex: 'p1', next: [15, 5, 0], prev: [5, 5, 0] },
        h2: { vertex: 'p2', prev: [15, -5, 0] }
    },
    curves: {
        c0: { from: 'p0', to: 'p1' },
        c1: { from: 'p1', to: 'p2' }
    },
    cyclic: false,
    radial_resolution: 8,
    tubular_resolution: 16,
    render_mode: 'mesh'
}).init();
```

## 12. Outliner 命名空间

`Outliner` 是 Blockbench 提供的全局命名空间，用于访问和操作大纲树（Outliner Tree）的结构与状态。

### 属性

| 属性名 | 类型 | 说明 |
|--------|------|------|

| `Outliner.elements` | `OutlinerElement[]` | 大纲中所有元素（可渲染的几何体） |
| `Outliner.selected` | `OutlinerElement[]` | 当前选中的元素 |
| `Outliner.nodes` | `OutlinerNode[]` | 大纲中所有节点 |

### OutlinerToggle 接口

大纲中的切换按钮定义接口，用于控制节点的显示属性。

```javascript
OutlinerToggle = {
    /** 按钮唯一标识 */
    id: string,

    /** 鼠标悬停提示标题 */
    title: string,

    /** 开启状态图标 */

    /** 关闭状态图标（可选） */
```

`Outliner.buttons` 中内置的按钮包括：

| 按钮 | 说明 |
|------|------|

| `export` | 导出开关 |
| `locked` | 锁定开关 |
| `mirror_uv` | UV 镜像切换 |
| `shade` | 着色切换 |
| `visibility` | 可见性切换 |

### 方法

**`Outliner.toJSON()`**
将大纲树序列化为 JSON 数组，用于持久化保存。

```javascript
const outlinerData = Outliner.toJSON();
// 保存到文件或用于撤销快照
```

**`Outliner.loadJSON(array, add_to_project?)`**
从 JSON 数组反序列化恢复大纲树。`add_to_project` 为 `true` 时，加载的节点自动加入当前项目。

```javascript
const savedOutliner = [...] // 之前保存的 JSON
```

**`Outliner.isNodeDisplayed(node)`**
检查指定节点是否在当前视图中显示（根据显示规则过滤）。

```javascript
if (Outliner.isNodeDisplayed(someNode)) {

```

**`Outliner.updateNodeDisplayRules()`**
更新所有节点的显示规则，在格式或模式变更后调用，刷新大纲显示状态。

## 13. Undo 系统

Blockbench 的撤销/重做系统，基于编辑条目（UndoEntry）机制，支持分组撤销操作。

### UndoSystem 类

#### 基本用法

撤销操作遵循 **"开始编辑 → 修改 → 完成编辑"** 三段式模式：

```javascript
// 第一步：开始编辑，记录初始状态
Undo.initEdit({ elements: [cube] });

// 第二步：执行修改
cube.name = '新名称';
cube.from = [-4, 0, -4];
cube.to = [4, 8, 4];

// 第三步：完成编辑，绑定操作名称
Undo.finishEdit('重命名立方体');
```

#### 方法

**`Undo.initEdit(aspects, amended?)`**
开始一次新的编辑操作，记录相关方面的初始状态快照。返回 `UndoEntry` 实例。

- **`aspects`**：`UndoAspects` 对象，指定需要记录的方面
- **`amended`**：可选，是否为修正编辑

**`Undo.finishEdit(action, aspects?)`**
完成编辑操作，将操作记录到撤销栈。`action` 为操作名称（显示在编辑菜单中）。返回 `UndoEntry` 实例。

**`Undo.cancelEdit()`**
取消编辑，恢复到初始状态。撤销栈不发生变化。

**`Undo.undo()`**
执行撤销操作，恢复到上一次编辑前的状态。

**`Undo.redo()`**
执行重做操作，恢复被撤销的编辑。

**`Undo.amendEdit(form, callback)`**
修改上一次编辑操作的信息。`form` 为表单对象，`callback` 为执行回调。

```javascript
Undo.edit(
    undo.custom ? '撤销：自定义' : '撤销：编辑',
    { elements: [cube], selection: true }
);
```

### UndoAspects 接口

`UndoAspects` 接口定义了撤销操作需要记录哪些方面的状态：

| 字段 | 类型 | 说明 |
|------|------|------|
| `selection` | `boolean` | 是否记录元素选择状态 |
| `elements` | `OutlinerElement[]` | 需要记录状态的特定元素列表 |
| `outliner` | `boolean` | 是否记录整个大纲结构 |
| `groups` | `Group[]` | 需要记录状态的组列表 |
| `textures` | `Texture[]` | 需要记录状态的纹理列表 |
| `animations` | `_Animation[]` | 需要记录状态的动画列表 |
| `keyframes` | `_Keyframe[]` | 需要记录状态的关键帧列表 |
| `display_slots` | `string[]` | 需要记录状态的显示槽位列表 |

### 完整示例

```javascript
function resizeCube(cube, newFrom, newTo) {
    // 开始编辑，记录元素和选择状态
    Undo.initEdit({
        elements: [cube],
        selection: true
    });

    // 执行修改
    cube.from = newFrom;
    cube.to = newTo;

    // 完成编辑，绑定操作名
    Undo.finishEdit('调整立方体尺寸');
}

// 撤销时可以捕获异常
try {
    Undo.undo();
} catch (e) {
    console.error('撤销失败:', e);
}
```
