# 事件系统 API 文档

## 概述

Blockbench 的事件系统为核心模块提供了松耦合的通信机制。`Blockbench` 对象本身和 `Codec` 等子对象均继承自 `EventSystem` 基类，支持事件的注册、派发和移除。事件系统是插件与 Blockbench 交互的主要方式，插件通过监听事件来响应程序状态的变化。

---

## 1. EventSystem 基类

所有可派发事件的对象都继承自此类。位于 `js/util/event_system.ts`。

### 内部结构

```typescript
class EventSystem {
    events: Record<string, EventListener[]>
}
```

每个事件名称对应一个回调函数数组。事件名称可以包含空格，表示同时监听多个事件。

### EventSystem 方法

#### dispatchEvent(event_name, data)

派发事件。遍历所有注册的回调并按注册顺序依次调用，汇集所有返回值。

- **参数：** `event_name` — 事件名称；`data` — 传递给回调的数据
- **返回值：** 最后一个非 `undefined` 的回调返回值（`EventSystem` 的原始实现）

```javascript
eventSystem.dispatchEvent('my_event', { key: 'value' })
```

**注意：** `Blockbench.dispatchEvent` 的返回值行为有所不同——它返回所有回调返回值组成的数组（见下文）。

#### on(event_name, callback)

注册事件监听器。

- **参数：** `event_name` — 事件名称，可用空格分隔同时监听多个事件；`callback(data)` — 回调函数
- **返回值：** `Deletable` 对象，调用其 `delete()` 方法可移除监听

```javascript
var listener = eventSystem.on('my_event', function(data) {
    console.log(data)
})
// 移除此监听
listener.delete()
```

**事件名称包含空格时**，会自动拆分为多个事件名并为每个名称注册同一回调，返回的 `Deletable` 可以一键移除所有注册。

#### once(event_name, callback)

注册一次性事件监听器。触发一次后自动移除。

- **参数：** `event_name` — 事件名称；`callback(data)` — 回调函数
- **返回值：** `Deletable` 对象

```javascript
eventSystem.once('load_project', function(data) {
    console.log('项目首次加载完成')
})
```

#### addListener(event_name, callback)

添加事件监听的别名，内部调用 `this.on(event_name, cb)`。

```javascript
eventSystem.addListener('update_selection', function(data) {
    // 处理选择更新
})
```

#### removeListener(event_name, callback)

移除事件监听器。支持空格分隔的多个事件名：

```javascript
eventSystem.removeListener('my_event', myCallback)
eventSystem.removeListener('event_a event_b', multiCallback)
```

### Deletable 接口

```typescript
type Deletable = {
    delete(): void
}
```

`on()` 方法返回 `Deletable` 对象，用于在不再需要时移除监听器。这是一种安全且便捷的监听管理方式，尤其适合插件在卸载时清理资源。

## 2. Blockbench 事件机制

`Blockbench` 对象的事件方法委托至 `EventSystem.prototype`，但 `dispatchEvent` 有独立的实现（见 [api.ts](file:///d:/Files/Project_MinecraftMods/blockbench/js/api.ts)）。

### Blockbench.dispatchEvent(event_name, data)

派发事件到所有已注册的监听器。与 `EventSystem` 的原始实现不同，此方法会：

1. 遍历所有注册的回调并依次调用，将返回值收集到数组中
2. 检查 `Validator` 是否订阅了此事件，若是则执行验证

- **返回值：** `any[]` — 所有回调返回值的数组

```javascript
let results = Blockbench.dispatchEvent('load_project', { model: data, path: filePath })
```

### Blockbench.on(event_name, callback)

注册事件监听器。通过 `EventSystem.prototype.on.call(this, ...)` 实现。

- **返回值：** `Deletable` — 调用 `delete()` 移除监听

```javascript
var listener = Blockbench.on('new_project', function(data) {
    console.log('新项目创建了', data)
})
```

### Blockbench.once(event_name, callback)

注册一次性事件监听器：

```javascript
Blockbench.once('load_project', function(data) {
    Blockbench.showQuickMessage('项目加载完成: ' + data.model.name)
})
```

### Blockbench.addListener(event_name, callback)

添加监听的别名：

```javascript
Blockbench.addListener('update_selection', function() {
    updateMyUI()
})
```

### Blockbench.removeListener(event_name, callback)

移除监听：

```javascript
Blockbench.removeListener('my_event', myHandler)
```

### 代码示例

```javascript
// 监听项目加载
Blockbench.on('load_project', function(data) {
    console.log('项目已加载', data.path)
})

// 监听选择变化
Blockbench.on('update_selection', function() {
    console.log('选择已更新')
})

// 监听按键
Blockbench.on('press_key', function(data) {
    if (data.event.key === 'Escape') {
        console.log('按下了 Escape 键')
    }
})
```

## 4. 最佳实践

### 插件中使用事件

```javascript
Plugin.register('my_plugin', {
    title: '我的插件',
    author: 'Me',
    icon: 'fa-plug',
    onload() {
        // 注册事件监听
        this.event_listeners = [
            Blockbench.on('new_project', function(data) {
                console.log('新项目创建')
            }),
            Blockbench.on('load_project', function(data) {
                console.log('项目加载:', data.path)
            }),
            Blockbench.on('update_selection', function() {
                console.log('选择更新')
            })
        ]
    },
    onunload() {
        // 清理所有事件监听
        if (this.event_listeners) {
            this.event_listeners.forEach(listener => listener.delete())
        }
    }
})
```

### 在 Codec 中使用事件

```javascript
var codec = new Codec('my_format', {
    name: '自定义格式',
    // ...
})

// 监听 codec 自有事件
codec.on('compile', function(data) {
    console.log('开始编译')
})

codec.once('parsed', function() {
    console.log('首次解析完成')
})
```

### 使用一次性事件

```javascript
// 仅在项目首次加载时执行
Blockbench.once('load_project', function(data) {
    Blockbench.showQuickMessage('欢迎！项目已加载')
})
```

### 通过返回值传递数据

```javascript
// 监听器可以返回数据给派发者
Blockbench.on('compile_texture_mcmeta', function(data) {
    if (myCondition) {
        data.mcmeta.custom_field = 'value'
    }
})
```

### 注意避免循环触发

事件监听器中应该避免派发相同的事件，以免造成无限循环。如果需要在事件处理中修改状态后再通知系统，使用不同的、更具体的事件名称。

## 5. 技术细节

### 事件注册的内部机制

`EventSystem` 使用一个 `Record<string, EventListener[]>` 对象存储事件监听器。每个事件名称对应一个回调函数数组。

```typescript
events = {
    'new_project': [cb1, cb2],
    'load_project': [cb3],
    'update_selection': [cb4, cb5, cb6]
}
```

### 支持多个事件名

`on()` 和 `removeListener()` 支持空格分隔的多个事件名：

```javascript
// 同时监听三个事件
Blockbench.on('new_project load_project close_project', function(data) {
    // 任何这三个事件触发时都会调用
})
```

### dispatchEvent 返回值差异

| 实现 | 返回值 |
|------|--------|
| `EventSystem.dispatchEvent` | 最后一个非 `undefined` 的回调返回值（`any`） |
| `Blockbench.dispatchEvent` | 所有回调返回值的数组（`any[]`），并在派发后触发 `Validator.validate` |

### 与 Validator 的集成

`Blockbench.dispatchEvent` 在派发事件后会检查 `Validator.triggers`，如果该事件在验证触发器列表中，会自动执行对应的验证逻辑。这是 Blockbench 条件系统与事件系统的集成点。
