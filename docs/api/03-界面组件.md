## 1. BarItem — 工具栏项基类

所有工具栏项（Action、Tool、Toggle、Widget 等）的基类。提供最基本的标识和 DOM 节点管理能力。

### 属性

| 属性 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `string` | 唯一标识符 |
| `name` | `string` | 显示名称 |
| `description` | `string` | 描述文本 |
| `icon` | `IconString` | 图标（支持 FontAwesome 类名如 `'fa-cube'`） |
| `condition` | `ConditionResolvable` | 条件表达式，控制项是否可用/可见 |
| `keybind` | `Keybind` | 关联的快捷键绑定 |
| `container` | `HTMLElement` | 容器 DOM 元素 |
| `nodes` | `HTMLElement[]` | 关联的 DOM 节点数组 |

### 方法

#### `barItem.getNode(): HTMLElement`

获取该 BarItem 对应的 DOM 节点。

```javascript
var actionNode = myAction.getNode();
document.body.appendChild(actionNode);
```

#### `barItem.delete(): void`

删除该 BarItem，从注册表移除并清理关联的 DOM 节点。

```javascript
myAction.delete();
```

#### `barItem.updateNode(): void`

更新 DOM 节点状态（如条件变化时刷新可用/禁用状态）。

## 2. Action — 可触发的动作

继承自 **BarItem(EventSystem)**。Action 是可被触发的动作，可添加到菜单和工具栏，可绑定快捷键。

### 构造方法

```javascript
new Action({
    id: 'my_action_id',           // 唯一标识
    name: '我的操作',              // 显示名称
    description: '操作描述',       // 描述文本
    icon: 'fa-cube',              // 图标（FontAwesome 类名）
    condition: ConditionResolvable, // 条件表达式（控制是否可用）
    keybind: new Keybind({key: 65, ctrl: true}), // 快捷键绑定
    click: function(event) {      // 点击回调函数
        Blockbench.showQuickMessage('触发了操作！');
    }
});
```

#### 构造参数详细说明

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | `string` | **是** | 全局唯一的标识符，用于注册表查找 |
| `name` | `string` | **是** | 向用户显示的友好名称 |
| `description` | `string` | 否 | 操作描述，常作为工具提示显示 |
| `icon` | `IconString` | 否 | 图标标识，如 `'fa-cube'`、`'fa-pencil'` |
| `condition` | `ConditionResolvable` | 否 | 条件表达式，为 `false` 时动作不可用 |
| `keybind` | `Keybind` | 否 | 绑定的快捷键 |
| `click` | `function` | **是** | 点击/触发时的回调，接收 `event` 参数 |

### 方法

#### `action.trigger(event?): void`

触发动作执行。调用后会执行 `click` 回调，并触发事件的 `dispatch` 传播。

```javascript
// 以编程方式触发动作
myAction.trigger();
```

#### `action.delete(): void`

删除动作。从 Action 注册表（Action.actions）中移除，同时清理所有引用和 DOM 节点。

```javascript
myAction.delete();
```

#### `action.setEnabled(enabled): void`

设置动作的启用/禁用状态，同时更新关联的 DOM 节点样式。

### 静态属性

| 属性 | 类型 | 说明 |
| --- | --- | --- |
| `Action.actions` | `object` | 所有已注册 Action 的映射表，以 `id` 为键 |
| `Action.all` | `Map` | 所有已注册 Action 的 Map |

### 完整示例

```javascript
// 定义一个完整的"导出 JSON"操作
var exportAction = new Action({
    id: 'mm_export_json',
    name: '导出 MachineMax JSON',
    description: '将当前项目的 MachineMax 配置导出为 JSON 文件',
    icon: 'fa-file-export',
    condition: function() {
        return Project && typeof Project.mm_config !== 'undefined';
    },
    keybind: new Keybind({key: 69, ctrl: true, shift: true}),
    click: function(event) {
        Blockbench.showQuickMessage('正在导出...');
        // 执行导出逻辑
    }
});

// 在其他地方通过 ID 触发
Action.actions.mm_export_json.trigger();
```

## 3. Tool — 工具

继承自 **Action**。工具类，如移动工具、画笔等。Tool 具有"选中/取消选中"的概念，同一时刻通常只有一个 Tool 被选中。

### 构造方法

```javascript
new Tool({
    id: 'my_tool',
    name: '我的工具',
    description: '这是一个自定义工具',
    icon: 'fa-pencil',
    // Action 原有属性
    click: function() { this.select(); },
    // Tool 特有属性
    onSelect: function() {
        // 工具被选中时调用
        console.log('工具已选中');
    },
    onUnselect: function() {
        // 工具被取消选中时调用
        console.log('工具已取消选中');
    }
});
```

#### 构造参数（Action 参数 + 以下特有参数）

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `onSelect` | `function` | 工具被选中时的回调 |
| `onUnselect` | `function` | 工具被取消选中时的回调 |
| `keybind` | `Keybind` | 通常绑定一个按键以便快速切换到此工具 |

### 方法

#### `tool.select(): void`

选中该工具。会自动通知工具箱（Toolbox）取消选中上一个工具，并触发 `onSelect` 回调。

```javascript
myTool.select();
```

#### `tool.trigger(): void`

触发工具。若工具未选中，则选中它；若已选中，则执行点击回调。

#### `tool.isSelected(): boolean`

检查该工具当前是否被选中。

### 示例

```javascript
// 定义自定义的"连接点标记"工具
var mmConnectorTool = new Tool({
    id: 'mm_connector_tool',
    name: '连接点标记',
    description: '在模型上标记连接点位置',
    icon: 'fa-crosshairs',
    keybind: new Keybind({key: 67}), // C 键
    onSelect: function() {
        // 进入连接点标记模式
        Blockbench.showQuickMessage('连接点标记模式已激活');
        Canvas.preview.style.cursor = 'crosshair';
    },
    onUnselect: function() {
        // 退出连接点标记模式
        Canvas.preview.style.cursor = 'default';
    },
    click: function(event) {
        this.select();
    }
});
```

## 4. Toggle — 开关

继承自 **Action**。开关型动作，具有开/关（on/off）两种状态。常用于控制某个模式或选项的开启与关闭。

### 构造方法

```javascript
new Toggle({
    id: 'my_toggle',
    name: '开关选项',
    description: '开启或关闭某项功能',
    icon: 'fa-toggle-on',
    condition: function() {
        return Project !== null;
    },
    click: function() {
        // 切换状态
        if (this.enabled) {
            Blockbench.showQuickMessage('已开启');
        } else {
            Blockbench.showQuickMessage('已关闭');
        }
    }
});
```

### 特有属性

| 属性 | 类型 | 说明 |
| --- | --- | --- |
| `toggle.enabled` | `boolean` | 当前开关状态（`true` = 开启） |

### 方法

#### `toggle.toggle(): void`

切换开关状态。自动切换 `enabled` 属性的值，触发 `click` 回调，并更新 DOM 节点的选中样式。

### 示例

```javascript
// 定义"显示连接点标签"的开关
var mmShowLabels = new Toggle({
    id: 'mm_show_labels',
    name: '显示连接点标签',
    description: '在 3D 视口中显示所有连接点的名称标签',
    icon: 'fa-tag',
    condition: function() {
        return Project && Project.mm_config;
    },
    click: function() {
        // 执行显示/隐藏逻辑
        updateConnectorLabels(this.enabled);
    }
});
```

## 5. Keybind — 快捷键绑定

管理键盘快捷键的绑定、录制和文本显示。

### 构造方法

```javascript
new Keybind({key: 83, ctrl: true, shift: false, alt: false});
```

#### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `key` | `number` | - | 键码（KeyboardEvent.keyCode / 字符码，如 83 = 'S'） |
| `ctrl` | `boolean` | `false` | 是否要求 Ctrl 键 |
| `shift` | `boolean` | `false` | 是否要求 Shift 键 |
| `alt` | `boolean` | `false` | 是否要求 Alt 键 |

### 方法

#### `keybind.set(keys): this`

设置按键。`keys` 可以是包含 `{key, ctrl, shift, alt}` 的对象，也可以是直接传入 `KeyCode`。

```javascript
keybind.set({key: 83, ctrl: true});
```

#### `keybind.clear(): this`

清除按键绑定。

```javascript
keybind.clear();
```

#### `keybind.save(): this`

将当前按键设置保存到 `localStorage`。

```javascript
keybind.save();
```

#### `keybind.setAction(id, sub_id?): this`

将当前快捷键绑定到指定的 Action。当快捷键被按下时，自动触发对应 Action。

```javascript
keybind.setAction('mm_export_json');
```

#### `keybind.getText(formatted?): string`

获取按键的人类可读文本表示。

```javascript
// 返回 "Ctrl+S"（formatted = true，带 HTML 格式）
// 返回 "Ctrl+S"（formatted = false，纯文本）
var displayText = keybind.getText(true);
```

#### `keybind.hasKey(): boolean`

检查是否已设置按键（非空绑定）。

```javascript
if (keybind.hasKey()) {
    console.log('快捷键已设置: ' + keybind.getText());
}
```

#### `keybind.isTriggered(event): boolean`

判断给定事件是否匹配当前的快捷键设置。

```javascript
document.addEventListener('keydown', function(event) {
    if (myKeybind.isTriggered(event)) {
        console.log('快捷键被触发！');
    }
});
```

#### `keybind.record(): this`

开始录制按键。调用后监听用户的下一次按键输入，自动设置按键组合。

```javascript
keybind.record();
```

#### `keybind.stopRecording(): this`

停止录制按键。

```javascript
keybind.stopRecording();
```

### 静态方法

#### `Keybind.loadKeymap(id, from_start_screen?): void`

加载指定 ID 的键位预设。

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `string` | 键位预设标识 |
| `from_start_screen` | `boolean` | 是否从启动界面加载 |

### 示例

```javascript
// 创建一个完整的快捷键绑定并关联到 Action
var keybind = new Keybind({key: 83, ctrl: true});
keybind.setAction('mm_export_json');
keybind.save();

// 在设置界面中让用户重新录制
$('#record-button').on('click', function() {
    keybind.record();
});
```

## 6. Toolbar — 工具栏

管理一组 BarItem 的容器，支持自定义布局、条件显示和用户配置保存。

### 构造方法

```javascript
new Toolbar('my_toolbar', {
    name: '我的工具栏',
    label: true,                  // 是否显示工具栏标签
    condition: function() {       // 条件：何时显示此工具栏
        return true;
    },
    children: [                   // 子项列表
        'my_action',              // 通过 ID 引用已注册的 Action
        '_',                      // 分隔符
        '+',                      // 空格
        'my_toggle'               // 通过 ID 引用已注册的 Toggle
    ]
});
```

#### 构造参数

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `name` | `string` | 工具栏 ID | 显示名称 |
| `label` | `boolean` | `false` | 是否显示标签 |
| `condition` | `function` | `() => true` | 条件函数 |
| `children` | `Array` | `[]` | 子项列表（Action ID、分隔符、空格） |

**children 特殊元素**：
- `'_'` — 分隔符（竖线）
- `'+'` — 空白间距

### 方法

#### `toolbar.add(action, position?): this`

向工具栏添加动作。

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `action` | `string` / `Action` | 动作 ID 或 Action 实例 |
| `position` | `number` | （可选）插入位置索引 |

```javascript
toolbar.add('mm_new_action');
toolbar.add(myAction, 0); // 插入到开头
```

#### `toolbar.remove(action): this`

从工具栏移除动作。

```javascript
toolbar.remove('mm_old_action');
```

#### `toolbar.update(): this`

更新工具栏的显示状态（如重新评估条件、刷新 DOM）。

```javascript
toolbar.update();
```

#### `toolbar.reset(): this`

重置工具栏为默认布局（清除用户的自定义修改）。

```javascript
toolbar.reset();
```

#### `toolbar.save(): this`

将当前工具栏配置保存到 `localStorage`，下次打开时自动恢复。

```javascript
toolbar.save();
```

#### `toolbar.condition(): boolean`

检查工具栏的显示条件。条件返回 `false` 时工具栏隐藏。

```javascript
if (toolbar.condition()) {
    toolbar.show();
}
```

### 示例

```javascript
// 创建 MachineMax 专属工具栏
var mmToolbar = new Toolbar('mm_toolbar', {
    name: 'MachineMax',
    label: true,
    condition: function() {
        return Project && Project.mm_config;
    },
    children: [
        'mm_add_part',
        '_',
        'mm_connector_tool',
        '+',
        'mm_show_labels',
        '_',
        'mm_export_json'
    ]
});
```

## 7. BARS 命名空间

全局的动作和工具栏管理命名空间，负责动作注册、工具栏初始化和条件更新。

### 属性

| 属性 | 类型 | 说明 |
| --- | --- | --- |
| `BARS.stored` | `object` | 存储在 localStorage 中的自定义工具栏配置 |

### 方法

#### `BARS.defineActions(definer): void`

定义/注册一组动作。`definer` 是一个回调函数，在其中创建 Action 实例。

```javascript
BARS.defineActions(function() {
    new Action({ id: 'mm_action_1', name: '动作1', click: handler1 });
    new Action({ id: 'mm_action_2', name: '动作2', click: handler2 });
    new Tool({ id: 'mm_tool_1', name: '工具1', icon: 'fa-wrench', onSelect: selectHandler });
});
```

#### `BARS.setupActions(): void`

设置所有已注册的动作。通常在插件初始化时调用，确保所有 Action 被正确初始化。

```javascript
// 在插件启动时调用
BARS.setupActions();
```

#### `BARS.setupToolbars(): void`

设置所有已注册的工具栏。通常在插件初始化时调用，构建工具栏 DOM 节点。

```javascript
BARS.setupToolbars();
```

#### `BARS.updateConditions(): void`

更新所有动作和工具栏的条件状态。当项目状态发生变化（如打开/关闭项目）时应调用此方法。

```javascript
// 项目状态变化时更新界面条件
Project.on('open', function() {
    BARS.updateConditions();
});
```

### Toolbox — 工具集合

Toolbox 是一个特殊的 Toolbar 实例，管理所有 Tool 类型。

| 属性 | 类型 | 说明 |
| --- | --- | --- |
| `Toolbox.selected` | `Tool` | 当前选中的工具 |

```javascript
// 获取当前选中的工具
var currentTool = Toolbox.selected;
console.log('当前工具: ' + currentTool.name);

// 通过 Toolbox 选中某个工具
Toolbox.selectTool('mm_connector_tool');
```

### 示例

```javascript
// 插件初始化的标准流程
function initPlugin() {
    // 1. 定义所有动作
    BARS.defineActions(function() {
        new Action({ id: 'mm_action', name: '动作', click: function() {} });
        new Tool({ id: 'mm_tool', name: '工具', onSelect: function() {} });
        new Toggle({ id: 'mm_toggle', name: '开关', click: function() {} });
    });

    // 2. 设置动作
    BARS.setupActions();

    // 3. 创建工具栏
    new Toolbar('mm_toolbar', {
        name: 'MachineMax',
        children: ['mm_action', '_', 'mm_tool', 'mm_toggle']
    });

    // 4. 设置工具栏
    BARS.setupToolbars();
}
```

## 8. Menu — 上下文菜单

上下文菜单，通常在右键点击时弹出，支持子菜单嵌套。

### 构造方法

```javascript
new Menu('my_menu', [
    {
        id: 'menu_item_1',
        name: '菜单项1',
        icon: 'fa-star',
        click: function() {
            Blockbench.showQuickMessage('点击了菜单项1');
        }
    },
    '_',  // 分隔符
    {
        name: '子菜单',
        icon: 'fa-folder',
        children: [
            {
                id: 'sub_item_1',
                name: '子菜单项',
                click: function() {
                    Blockbench.showQuickMessage('点击了子菜单项');
                }
            }
        ]
    }
], {
    onOpen: function(position, context) {
        console.log('菜单已打开', position, context);
    },
    onClose: function() {
        console.log('菜单已关闭');
    }
});
```

#### 构造参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `id` | `string` | **是** | 菜单唯一标识 |
| `menuItems` | `Array` | **是** | 菜单项数组（见下方说明） |
| `options` | `object` | 否 | 配置选项 |

**菜单项结构**：
| 属性 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `string` | 菜单项 ID |
| `name` | `string` | 显示名称 |
| `icon` | `string` | 图标类名 |
| `click` | `function` | 点击回调 |
| `children` | `Array` | 子菜单项数组 |
| `condition` | `function` | 条件函数 |
| `keybind` | `Keybind` | 显示的快捷键提示 |

**特殊字符串项**：
- `'_'` — 分隔符

### 方法

#### `menu.open(position, context?): this`

打开菜单。

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `position` | `MouseEvent` / `HTMLElement` / `'mouse'` | 打开位置 |
| `context` | `any` | （可选）上下文数据，可在菜单项回调中访问 |

```javascript
// 在鼠标位置打开
menu.open('mouse');

// 在指定元素位置打开
menu.open(someElement);

// 在鼠标事件位置打开
element.addEventListener('contextmenu', function(event) {
    event.preventDefault();
    contextMenu.open(event);
});
```

#### `menu.show(position, context?): this`

`open()` 的别名方法。

```javascript
menu.show('mouse');
```

#### `menu.hide(): this`

关闭菜单。

```javascript
menu.hide();
```

#### `menu.addAction(action, path?): void`

向菜单添加一个动作。支持通过 `path` 指定插入位置。

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `action` | `Action` | 要添加的动作 |
| `path` | `string` | （可选）路径，格式如 `'子菜单ID.0'` |

```javascript
// 添加到根菜单
menu.addAction(myAction);

// 添加到"导出"子菜单的顶部位置
menu.addAction(myAction, 'export.0');

// 添加到"导出"子菜单的第二个位置
menu.addAction(myAction, 'export.1');
```

#### `menu.removeAction(path): void`

从菜单移除指定路径的动作。

```javascript
// 移除根菜单中的动作
menu.removeAction('my_action');

// 移除子菜单中的动作
menu.removeAction('export.my_action');
```

#### `menu.highlight(action): void`

高亮显示菜单中的某个动作（仅 BarMenu 有效）。

### 静态属性

| 属性 | 类型 | 说明 |
| --- | --- | --- |
| `Menu.open` | `Menu` | 当前处于打开状态的菜单，`null` 表示没有打开的菜单 |

### 完整示例

## 11. Dialog — 对话框

模态对话框，支持表单、自定义按钮和多种回调。

### 构造方法

```javascript
var dialog = new Dialog({
    id: 'my_dialog',              // 唯一标识
    title: '我的对话框',            // 标题
    width: 400,                   // 宽度（像素）
    height: 300,                  // 高度（像素）
    lines: [                      // 行内容数组（文本或 DOM）
        '请输入以下信息：',
        { text: '提示文本', type: 'info' }
    ],
    form: {                       // 表单配置（见 Form 章节）
        name: {
            label: '名称',
            type: 'text',
            value: '默认值'
        }
    },
    buttons: ['确认', '取消'],     // 按钮文本数组
    onConfirm: function(result) {  // 确认回调
        console.log('用户确认', result);
    },
    onCancel: function() {        // 取消回调
        console.log('用户取消');
    },
    onOpen: function() {          // 打开回调
        console.log('对话框已打开');
    }
});

dialog.show();
```

#### 构造参数

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `id` | `string` | - | 唯一标识（用于 CSS 和引用） |
| `title` | `string` | `''` | 对话框标题 |
| `width` | `number` | `auto` | 宽度（像素） |
| `height` | `number` | `auto` | 高度（像素） |
| `lines` | `Array` | `[]` | 文本行或 DOM 元素的混合数组 |
| `form` | `object` | `null` | InputForm 配置对象 |
| `buttons` | `Array` | `['确认']` | 按钮文本数组 |
| `onConfirm` | `function` | `null` | 确认回调 |
| `onCancel` | `function` | `null` | 取消回调 |
| `onOpen` | `function` | `null` | 打开回调 |

### 方法

#### `dialog.show(): void`

显示对话框。第一次显示时自动构建 DOM。

```javascript
dialog.show();
```

#### `dialog.hide(): void`

隐藏对话框（不销毁）。

```javascript
dialog.hide();
```

#### `dialog.build(): void`

构建对话框的 DOM 结构。通常在第一次 `show()` 时自动调用。

```javascript
dialog.build();
```

#### `dialog.close(): void`

关闭并销毁对话框。

```javascript
dialog.close();
```

#### `dialog.confirm(result?): void`

确认对话框。执行 `onConfirm` 回调并关闭对话框。

```javascript
dialog.confirm({ name: '测试', count: 5 });
```

#### `dialog.cancel(): void`

取消对话框。执行 `onCancel` 回调并关闭对话框。

```javascript
dialog.cancel();
```

#### `dialog.focus(): void`

聚焦对话框（使其获得键盘焦点）。

```javascript
dialog.focus();
```

## 15. 工具栏控件 (Widget)

工具栏中可使用的各种交互控件。

### NumSlider — 数值滑块

数值滑块控件，带有输入框和滑条，可拖拽或输入精确数值。

```javascript
new NumSlider({
    id: 'my_slider',
    name: '数值',
    value: 0,
    min: -180,
    max: 180,
    step: 1
});
```

#### 构造参数

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `id` | `string` | - | 唯一标识 |
| `name` | `string` | `''` | 显示名称 |
| `value` | `number` | `0` | 初始值 |
| `min` | `number` | `0` | 最小值 |
| `max` | `number` | `100` | 最大值 |
| `step` | `number` | `1` | 步进值 |

```javascript
// 创建旋转角度滑块
var rotationSlider = new NumSlider({
    id: 'mm_rotation',
    name: '旋转角度',
    value: 0,
    min: -360,
    max: 360,
    step: 5
});

// 监听值变化（如果有回调属性）
// 注意：NumSlider 继承自 BarItem，可通过 EventSystem 监听
```

### BarSelect — 下拉选择器

工具栏中的下拉选择控件。

```javascript
new BarSelect({
    id: 'my_select',
    label: '选项',
    options: {
        option1: '选项1',
        option2: '选项2',
        option3: '选项3'
    }
});
```

#### 构造参数

| 参数 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `id` | `string` | - | 唯一标识 |
| `label` | `string` | `''` | 标签文本 |
| `options` | `object` | `{}` | 选项映射 `{ value: displayText }` |
| `value` | `string` | - | 当前选中的值 |

```javascript
// 创建零件类型选择器
var partTypeSelect = new BarSelect({
    id: 'mm_part_type',
    label: '类型',
    options: {
        body: '车身',
        wheel: '车轮',
        weapon: '武器',
        accessory: '装饰'
    },
    value: 'body'
});

// 监听选择变化
partTypeSelect.on('change', function(value) {
    console.log('选中了: ' + value);
});
```

## 附录：最佳实践

### 1. ID 命名规范

为避免与其它插件的 ID 冲突，建议使用插件专属前缀：

```
mm_          → MachineMax 插件
mm_action_   → MachineMax 动作
mm_tool_     → MachineMax 工具
mm_panel_    → MachineMax 面板
```

### 2. 条件表达式的使用

条件函数应始终返回布尔值，且不产生副作用：

```javascript
// 推荐的写法
condition: function() {
    return Project && Project.mm_config;
}

// 避免在 condition 中修改状态
condition: function() {
    // 错误：不要在 condition 中修改状态
    updateSomething();
    return true;
}
```

### 3. 生命周期管理

```javascript
// 插件启用时
function onPluginLoad() {
    BARS.defineActions(function() {
        // 注册动作
    });
    BARS.setupActions();
    BARS.setupToolbars();
}

// 插件禁用时
function onPluginUnload() {
    // 清理 Action
    Object.values(Action.actions).forEach(function(action) {
        if (action.id.startsWith('mm_')) {
            action.delete();
        }
    });

    // 清理工具栏
    // 清理面板
    Object.keys(Panels).forEach(function(key) {
        if (key.startsWith('mm_')) {
            Panels[key].delete();
        }
    });
}
```

### 4. 界面状态更新

```javascript
// 在项目打开/关闭时更新界面
Project.on('open', function() {
    BARS.updateConditions();
    Toolbox.update();
    MenuBar.update();
});

Project.on('close', function() {
    BARS.updateConditions();
    Toolbox.update();
    MenuBar.update();
});
```
