## 1. _Animation 类 — 动画

> **注意**：因 TypeScript 无法覆盖 UMD 全局类型，核心类以 `_Animation` 命名。在 Blockbench 内部及插件开发中均使用此名称。

`_Animation` 是 Blockbench 动画系统的核心类，代表一个独立的动画片段。每个动画包含名称、时长、循环模式以及一组动画器（Animator），动画器驱动骨骼、空对象或效果的变化。

### 构造函数

```javascript
new Animation({
    name: '动画名称',
    length: 2.0,
    loop: 'loop',       // 'once' | 'hold' | 'loop'
    snapping: 24,       // 每秒帧数
    animators: {}       // 动画器字典
});
```

#### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `name` | `string` | — | 动画名称，显示在时间轴和动画列表中 |
| `length` | `number` | — | 动画总长度，单位：秒 |
| `loop` | `'once' \| 'hold' \| 'loop'` | `'loop'` | 循环模式 |
| `snapping` | `number` | `24` | 每秒帧数（对齐精度），决定关键帧吸附粒度 |
| `animators` | `Record<string, GeneralAnimator>` | `{}` | 动画器字典，键为动画器 uuid，值为动画器实例 |

#### 循环模式说明

| 模式 | 说明 |
|------|------|
| `'once'` | 播放一次后停止，停留在最后一帧 |
| `'hold'` | 播放一次后保持最后一帧的状态 |
| `'loop'` | 循环播放，结束后从开头重新播放 |

### 关键属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `name` | `string` | 动画名称，显示在 UI 中 |
| `uuid` | `UUID` | 动画唯一标识符 |
| `loop` | `'once' \| 'hold' \| 'loop'` | 循环模式 |
| `override` | `boolean` | 是否覆盖其他同名动画（用于基岩版导出） |
| `length` | `number` | 动画长度，单位：秒 |
| `snapping` | `number` | 每秒帧数，控制关键帧吸附步长 |
| `playing` | `boolean` | 是否正在播放 |
| `time` | `number` | 当前播放时间位置 |
| `markers` | `TimelineMarker[]` | 时间轴标记列表 |
| `animators` | `Record<string, GeneralAnimator>` | 动画器字典（键为 uuid），驱动被动画对象 |
| `selected` | `boolean` | 是否在动画列表中选中 |

### 关键方法

#### **`animation.select()`**
选中当前动画，在动画列表中高亮显示。

```javascript
// 选中项目中的第一个动画
Project.animations[0].select();
```

#### **`animation.remove()`**
从项目中移除当前动画及其所有关键帧数据。

```javascript
// 移除指定动画
const anim = Project.animations.find(a => a.name === 'walk');
if (anim) anim.remove();
```

#### **`animation.rename()`**
进入重命名模式，弹出内联编辑框让用户输入新名称。

```javascript
// 触发重命名 UI
animation.rename();
```

#### **`animation.duplicate()`**
复制当前动画，返回新的 `_Animation` 实例。复制体包含相同的名称、时长、循环模式以及所有动画器和关键帧数据。

```javascript
// 复制动画
const copy = animation.duplicate();
Project.animations.push(copy);
```

#### **`animation.propertiesDialog()`**
打开动画属性设置对话框，允许用户编辑名称、长度、循环模式等属性。

```javascript
// 打开属性对话框
animation.propertiesDialog();
```

#### **`animation.getUndoCopy()`**
获取当前动画的快照数据，用于撤销操作。返回 `AnimationUndoCopy` 对象。

```javascript
// 在编辑操作前保存快照
const undoCopy = animation.getUndoCopy();

// 执行修改
animation.name = '新名称';
animation.length = 3.0;

// 撤销时恢复到 undoCopy 的状态
```

### 完整示例

```javascript
// 创建一个行走动画
const walkAnim = new Animation({
    name: 'walk',
    length: 1.0,
    loop: 'loop',
    snapping: 24,
    animators: {}
});

// 将动画添加到项目
Project.animations.push(walkAnim);

// 为动画创建骨骼动画器并添加关键帧
const boneAnimator = new BoneAnimator({
    name: 'left_leg',
    target: leftLegBone
});

// 添加位置关键帧
const posKeyframe = new Keyframe({
    channel: 'position',
    time: 0.0,
    data_points: [{ x: 0, y: 0, z: 0 }],
    interpolation: 'linear'
});
boneAnimator.channels.push(posKeyframe);

// 将动画器添加到动画字典中
walkAnim.animators[boneAnimator.uuid] = boneAnimator;
```

## 2. _Keyframe 类 — 关键帧

`_Keyframe` 表示动画时间轴上的一个关键帧，定义了某个属性在特定时间点的值。多个关键帧通过插值形成连续动画。

### 构造函数

构造函数接收三个参数：`data`（关键帧数据对象）、`uuid`（可选 UUID）、`animator`（可选所属动画器引用）。

```javascript
new Keyframe({
    channel: 'position',    // 'position' | 'rotation' | 'scale'
    time: 0.5,
    data_points: [{ x: 0, y: 0, z: 0 }],
    interpolation: 'linear' // 'linear' | 'catmullrom' | 'bezier' | 'step'
});
```

#### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `channel` | `string` | — | 通道名，指定动画属性类型 |
| `time` | `number` | — | 关键帧时间位置，单位：秒 |
| `data_points` | `KeyframeDataPoint[]` | — | 数据点数组，每个数据点包含变换值 |
| `interpolation` | `string` | `'linear'` | 插值方式 |

#### 通道（Channel）说明

| 通道名 | 适用动画器 | 说明 |
|--------|-----------|------|
| `'position'` | `BoneAnimator`, `NullObjectAnimator` | 位置变换（位移） |
| `'rotation'` | `BoneAnimator`, `NullObjectAnimator` | 旋转变换 |
| `'scale'` | `BoneAnimator`, `NullObjectAnimator` | 缩放变换 |
| `'effect'` | `EffectAnimator` | 效果触发（粒子、声音等） |

#### 插值方式说明

| 方式 | 枚举值 | 说明 |
|------|--------|------|
| 线性插值 | `'linear'` | 两个关键帧之间匀速过渡，最常用 |
| Catmull-Rom 样条 | `'catmullrom'` | 平滑曲线插值，通过所有关键帧点，生成流畅曲线 |
| 贝塞尔插值 | `'bezier'` | 贝塞尔曲线插值，通过控制点调节曲线形态 |
| 阶跃插值 | `'step'` | 无过渡，到达关键帧时间时瞬间跳变到目标值 |

### 关键属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `data_points` | `KeyframeDataPoint[]` | 数据点数组，每个数据点为一个三维值 `{x, y, z}` |
| `animator` | `GeneralAnimator` | 所属的动画器引用 |
| `channel` | `string` | 通道名：`'position'` / `'rotation'` / `'scale'` |
| `time` | `number` | 关键帧在时间轴上的位置，单位：秒 |
| `uuid` | `string` | 关键帧唯一标识符 |
| `interpolation` | `string` | 插值方式：`'linear'` / `'catmullrom'` / `'bezier'` / `'step'` |
| `bezier_linked` | `boolean` | 贝塞尔控制点是否关联（左右控制点联动） |
| `bezier_left_time` | `ArrayVector3` | 左侧贝塞尔时间控制点 |
| `bezier_right_time` | `ArrayVector3` | 右侧贝塞尔时间控制点 |
| `bezier_left_value` | `ArrayVector3` | 左侧贝塞尔值控制点 |
| `bezier_right_value` | `ArrayVector3` | 右侧贝塞尔值控制点 |
| `selected` | `boolean` | 是否在时间轴中被选中 |
| `has_expressions` | `boolean` | 是否包含 Molang 表达式（基岩版） |

### 关键方法

#### **`keyframe.get(axis, data_point?)`**
获取指定轴的值。`axis` 为 `'x'` / `'y'` / `'z'`。可通过 `data_point` 指定数据点索引，默认为 0。返回 `number` 或表达式字符串。

```javascript
const x = keyframe.get('x');     // 获取 X 轴值
const y = keyframe.get('y', 1);  // 获取第二个数据点的 Y 轴值
```

#### **`keyframe.calc(axis, data_point?)`**
获取计算后的数值。与 `get()` 的区别在于：如果值是表达式，`calc()` 会解析并计算表达式，始终返回 `number`。

```javascript
// 假设关键帧的值是表达式 "math.sin(q.anim_time)"
const value = keyframe.calc('x'); // 返回解析计算后的数值
```

#### **`keyframe.set(axis, value, data_point?)`**
设置指定轴的值。`data_point` 指定数据点索引，默认为 0。返回 `this` 支持链式调用。

```javascript
keyframe.set('x', 10.5)
       .set('y', 0.0, 1);  // 设置第二个数据点的 Y 轴值
```

#### **`keyframe.offset(axis, amount)`**
对指定轴的值进行偏移增加。

```javascript
// 将 X 轴值偏移 2 个单位
keyframe.offset('x', 2);
```

#### **`keyframe.flip(axis)`**
翻转指定轴的值（取反）。返回 `this` 支持链式调用。

```javascript
// 翻转 Z 轴值（例如将位移方向取反）
keyframe.flip('z');
```

#### **`keyframe.getLerp(other, axis, amount)`**
计算当前关键帧与另一个关键帧之间的线性插值结果。`amount` 为插值因子（0~1）。

```javascript
const startKey = new Keyframe({ channel: 'position', time: 0, data_points: [{ x: 0, y: 0, z: 0 }] });
const endKey   = new Keyframe({ channel: 'position', time: 1, data_points: [{ x: 10, y: 0, z: 0 }] });

// 计算 50% 位置的插值
const mid = startKey.getLerp(endKey, 'x', 0.5);
// 返回: 5
```

#### **`keyframe.getCatmullromLerp(before_plus, before, after, after_plus, axis, alpha)`**
Catmull-Rom 样条插值计算。需要传入前两个、后两个共四个相邻关键帧作为控制点。

| 参数 | 说明 |
|------|------|
| `before_plus` | 当前关键帧的前两个关键帧 |
| `before` | 当前关键帧的前一个关键帧 |
| `after` | 当前关键帧的后一个关键帧 |
| `after_plus` | 当前关键帧的后两个关键帧 |
| `axis` | 坐标轴（`'x'` / `'y'` / `'z'`） |
| `alpha` | 插值参数（与 `amount` 类似） |

```javascript
// 计算 Catmull-Rom 插值（通常由 Blockbench 内部调用，此处仅为示例）
const value = keyframe.getCatmullromLerp(kf0, kf1, kf2, kf3, 'y', 0.5);
```

#### **`keyframe.getFixed(data_point?)`**
获取格式固定的数据值。根据通道类型自动返回对应格式：
- `'position'` → `Vector3`
- `'rotation'` → `Euler`
- `'scale'` → `Vector3`

```javascript
const pos = keyframe.getFixed(); // 返回 Vector3 或 Euler
```

#### **`keyframe.select(event?)`**
选中关键帧。传入 `event` 参数时可配合 Shift/Ctrl 实现多选。返回 `this`。

```javascript
// 选中单个关键帧
keyframe.select();

// 带事件的选择（支持多选修饰键）
keyframe.select(mouseEvent);
```

#### **`keyframe.remove()`**
从所属动画器中移除关键帧。

```javascript
// 移除关键帧
keyframe.remove();
```

#### **`keyframe.showContextMenu(event)`**
在鼠标事件位置显示关键帧右键菜单。返回 `this`。

```javascript
// 右键菜单
keyframe.showContextMenu(rightClickEvent);
```

### 完整示例

```javascript
// 创建一组旋转关键帧，实现 0° → 90° → 0° 的循环摆动
const kf0 = new Keyframe({
    channel: 'rotation',
    time: 0.0,
    data_points: [{ x: 0, y: 0, z: 0 }],
    interpolation: 'linear'
});

const kf1 = new Keyframe({
    channel: 'rotation',
    time: 0.5,
    data_points: [{ x: 0, y: 90, z: 0 }],
    interpolation: 'linear'
});

const kf2 = new Keyframe({
    channel: 'rotation',
    time: 1.0,
    data_points: [{ x: 0, y: 0, z: 0 }],
    interpolation: 'linear'
});

// 添加到动画器
boneAnimator.channels.push(kf0, kf1, kf2);

// 修改关键帧值
kf1.set('y', 45); // 将旋转改为 45°
```

## 3. AnimationItem 类 — 动画项目基类

`AnimationItem` 是所有可动画项目（Animation、AnimationController 等）的抽象基类。提供了统一的静态访问接口。

### 静态属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `AnimationItem.all` | `[]` | 所有动画项目实例数组 |
| `AnimationItem.selected` | `[]` | 当前选中的动画项目数组 |

### 方法

#### **`getUndoCopy()`**
获取用于撤销操作的拷贝数据。子类需要实现此方法以返回正确的快照格式。

```javascript
const copy = animationItem.getUndoCopy();
// 修改后，可通过 copy 执行撤销恢复
```

## 4. AnimationController 类 — 动画控制器（基岩版）

`AnimationController` 对应 Minecraft 基岩版（Bedrock Edition）的动画状态机系统。每个控制器包含一组状态（State），状态之间通过过渡（Transition）相互切换，实现复杂的动画逻辑。

### 构造函数

```javascript
new AnimationController({
    name: '控制器名称',
    initial_state: 'default'
});
```

#### 参数说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `name` | `string` | — | 控制器名称，用于导出标识 |
| `initial_state` | `string` | `'default'` | 初始状态的名称 |

### 关键属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `name` | `string` | 控制器名称 |
| `path` | `string` | 控制器在项目中的路径 |
| `states` | `AnimationControllerState[]` | 状态列表，每个状态包含动画和过渡规则 |
| `initial_state` | `string` | 初始状态名称，指定实体加载时默认进入的状态 |
| `selected_state` | `AnimationControllerState \| null` | 当前在 UI 中选中的状态 |
| `playing` | `boolean` | 是否正在预览播放 |

### 关键方法

#### **`controller.add(undo?)`**
将控制器添加到当前项目中。可选 `undo` 参数控制是否可撤销。

```javascript
const ctrl = new AnimationController({
    name: 'movement',
    initial_state: 'idle'
});
ctrl.add(); // 加入项目
```

#### **`controller.remove(undo?)`**
从项目中移除控制器。可选 `undo` 参数控制是否可撤销。

```javascript
// 移除控制器
controller.remove();
```

#### **`controller.propertiesDialog()`**
打开控制器属性设置对话框。

```javascript
controller.propertiesDialog();
```

#### **`controller.updatePreview()`**
更新控制器的 3D 视口预览。

```javascript
// 修改状态后刷新预览
controller.updatePreview();
```

#### **`controller.compileForBedrock()`**
将控制器编译为基岩版 JSON 格式，用于 Minecraft 行为包。

```javascript
// 编译控制器
const bedrockData = controller.compileForBedrock();
// 输出示例：
// {
//   "format_version": "1.17.30",
//   "animation_controllers": {
//     "controller.movement": {
//       "initial_state": "idle",
//       "states": { ... }
//     }
//   }
// }
```

### 完整示例

```javascript
// 创建一个基岩版动画控制器
const ctrl = new AnimationController({
    name: 'controller.character',
    initial_state: 'idle'
});
ctrl.add();

// 创建状态
const idleState = new AnimationControllerState({
    name: 'idle',
    animations: [{ animation: 'animation.character.idle' }],
    blend_transition: 0.25
});

const walkState = new AnimationControllerState({
    name: 'walk',
    animations: [{ animation: 'animation.character.walk' }],
    blend_transition: 0.15
});

// 添加状态间的过渡
idleState.addTransition(walkState.uuid, {
    condition: 'query.modified_move_speed > 0.1'
});

walkState.addTransition(idleState.uuid, {
    condition: 'query.modified_move_speed <= 0.1'
});

// 将状态添加到控制器
ctrl.states.push(idleState, walkState);
```

## 5. AnimationControllerState 类 — 动画控制器状态

`AnimationControllerState` 表示控制器的单个状态。每个状态包含一个或多个动画、过渡规则、音效和粒子效果。

### 构造函数

```javascript
new AnimationControllerState({
    name: '状态名称',
    animations: [],
    transitions: [],
    sounds: [],
    particles: [],
    on_entry: '',
    on_exit: '',
    blend_transition: 0.0,
    blend_via_shortest_path: false
});
```

### 关键属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `controller` | `AnimationController` | 所属的控制器的引用 |
| `name` | `string` | 状态名称，同控制器内必须唯一 |
| `animations` | `StateAnimation[]` | 状态播放的动画列表 |
| `transitions` | `any[]` | 过渡规则列表，定义何时切换到其他状态 |
| `sounds` | `any[]` | 音效列表，进入状态时播放 |
| `particles` | `any[]` | 粒子列表，进入状态时生成 |
| `on_entry` | `string` | 进入状态时执行的命令（Molang / 斜杠命令） |
| `on_exit` | `string` | 离开状态时执行的命令 |
| `blend_transition` | `number` | 混合过渡时间，单位：秒 |
| `blend_via_shortest_path` | `boolean` | 是否通过最短路径混合旋转（避免绕大圈） |

### 关键方法

#### **`state.select(force?)`**
在 UI 中选中该状态。`force` 为 `true` 时强制选中。

```javascript
state.select();
```

#### **`state.remove(undo?)`**
从控制器中移除该状态。可选 `undo` 控制是否可撤销。

```javascript
state.remove();
```

#### **`state.addAnimation(animation?)`**
向状态中添加动画。可传入动画名称或 `_Animation` 实例。

```javascript
state.addAnimation('animation.character.walk');
```

#### **`state.addTransition(target_uuid?)`**
向状态添加过渡规则。`target_uuid` 为目标状态的 UUID，不传时打开选择对话框。

```javascript
state.addTransition(targetState.uuid, {
    condition: 'query.is_sneaking'
});
```

#### **`state.addParticle(options?)`**
添加粒子效果，进入状态时生成。

```javascript
state.addParticle({
    effect: 'minecraft:critical_hit_emitter',
    locator: 'body'
});
```

#### **`state.addSound(options?)`**
添加音效，进入状态时播放。

```javascript
state.addSound({
    sound: 'step.grass',
    volume: 1.0,
    pitch: [0.8, 1.2]
});
```

#### **`state.compileForBedrock()`**
将状态编译为基岩版 JSON 格式。

```javascript
const stateData = state.compileForBedrock();
// 输出：
// {
//   "animations": ["animation.character.walk"],
//   "transitions": [...],
//   "on_entry": [...],
//   "particle_effects": [...],
//   "sound_effects": [...]
// }
```

## 6. Animator 命名空间 — 全局动画器

`Animator` 是 Blockbench 的全局动画器命名空间，管理所有动画的播放状态和预览。**注意：Animator 不是时间轴控制器，实际的播放控制由 `Timeline` 管理。**

### 全局属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `Animator.open` | `boolean` | 动画模式是否打开 |
| `Animator.animations` | `Animation[]` | 所有动画实例（只读，委托至 `Animation.all`） |
| `Animator.selected` | `Animation` | 当前选中的动画（只读，委托至 `Animation.selected`） |
| `Animator.MolangParser` | `MolangParser` | Molang 表达式解析器实例 |
| `Animator.motion_trail` | `THREE.Object3D` | 运动轨迹辅助对象 |
| `Animator.onion_skin_object` | `THREE.Object3D` | 洋葱皮辅助对象 |

### 方法

#### **`Animator.preview(in_loop?)`**
执行全局动画预览。在动画播放循环中每帧调用，更新所有动画器的状态到场景。

```javascript
Animator.preview(true);  // 在动画循环中调用
```

#### **`Animator.showDefaultPose(reduced_updates?)`**
显示所有骨骼的默认姿势（不受动画影响）。

```javascript
Animator.showDefaultPose();
```

#### **`Animator.resetParticles(optimized?)`**
重置所有粒子效果。

```javascript
Animator.resetParticles();
```

#### **`Animator.showMotionTrail(target?, fast?)`**
显示指定骨骼的运动轨迹。

```javascript
Animator.showMotionTrail(selectedBone);
```

#### **`Animator.updateOnionSkin()`**
更新洋葱皮辅助显示（用于动画对比参考帧）。

#### **`Animator.stackAnimations(animations, in_loop?, controller_blend_values?)`**
将多个动画叠加计算，处理混合叠加逻辑。

### 使用示例

```javascript
// 检查动画模式是否打开
if (Animator.open) {
    console.log('动画模式已激活');
}

// 获取当前选中的动画
var anim = Animator.selected;
console.log('当前动画:', anim.name);
```

## 7. GeneralAnimator 类 — 通用动画器基类

`GeneralAnimator` 是所有动画器的抽象基类，定义了动画器的通用接口。每个动画器负责驱动一个特定目标对象（骨骼、空对象等）的动画属性。

### 构造函数

```javascript
new GeneralAnimator(uuid, animation);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `uuid` | `string` | 动画器唯一标识符 |
| `animation` | `Animation` | 所属动画实例 |

### 关键属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `uuid` | `string` | 动画器 UUID |
| `animation` | `Animation` | 所属动画引用 |
| `channels` | `Channel[]` | 通道列表，每个通道对应一类变换属性 |
| `keyframes` | `object` (getter) | 关键帧映射对象，键为通道名，值为关键帧数组 |
| `muted` | `Record<string, boolean>` | 通道静音状态字典 |
| `expanded` | `boolean` | 在时间轴中是否展开 |
| `selected` | `boolean` | 是否被选中 |

### 方法

#### **`animator.addKeyframe(data, uuid?)`**
添加关键帧到动画器。

```javascript
var kf = boneAnim.addKeyframe({
    channel: 'position',
    time: 0.5,
    data_points: [{ x: 5, y: 0, z: 0 }]
});
```

#### **`animator.getOrMakeKeyframe(channel)`**
获取或创建指定通道的关键帧。返回 `{ before, result, new_keyframe }`。

#### **`animator.toggleMuted(channel)`**
切换指定通道的静音状态。

```javascript
boneAnim.toggleMuted('rotation');
```

### 关键帧与通道

`GeneralAnimator` 通过 `channels` 管理多个变换通道。可以通过 `GeneralAnimator.addChannel()` 静态方法注册新的通道类型：

```javascript
GeneralAnimator.addChannel('custom', {
    name: '自定义通道',
    transform: true,
    mutable: true,
    max_data_points: 1
});
```

## 8. BoneAnimator 类 — 骨骼动画器

`BoneAnimator` 继承自 `GeneralAnimator`，专门用于驱动 `Group`（骨骼）对象的变换。它管理骨骼的旋转、位移和缩放三组关键帧。

### 构造函数

```javascript
new BoneAnimator(uuid, animation, name);
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `uuid` | `string` | 动画器 UUID |
| `animation` | `Animation` | 所属动画实例 |
| `name` | `string` | 骨骼名称（可选） |

### 功能说明

| 通道 | 变换类型 | 说明 |
|------|----------|------|
| `'position'` | 位移 | 骨骼位置偏移 |
| `'rotation'` | 旋转 | 骨骼欧拉旋转角度 |
| `'scale'` | 缩放 | 骨骼缩放比例 |

### 方法

#### **`animator.getGroup()`**
返回该动画器驱动的骨骼对象。

#### **`animator.displayFrame(multiplier?)`**
一次性应用旋转 + 位置 + 缩放变换到骨骼场景对象。

#### **`animator.interpolate(channel, allow_expression?, axis?)`**
计算指定通道在当前时间的插值结果。

### 使用示例

```javascript
// 创建骨骼动画器
const boneAnim = new BoneAnimator(
    'uuid-1234',
    walkAnim,
    'left_leg'
);

// 添加关键帧到通道
boneAnim.addKeyframe({
    channel: 'position',
    time: 0.0,
    data_points: [{ x: 0, y: 0, z: 0 }]
});
boneAnim.addKeyframe({
    channel: 'position',
    time: 0.5,
    data_points: [{ x: 5, y: 0, z: 0 }]
});
boneAnim.addKeyframe({
    channel: 'position',
    time: 1.0,
    data_points: [{ x: 0, y: 0, z: 0 }]
});

// 将动画器添加到动画字典中
walkAnim.animators[boneAnim.uuid] = boneAnim;
```

## 9. NullObjectAnimator 类 — 空对象动画器

`NullObjectAnimator` 继承自 `GeneralAnimator`，用于驱动 `NullObject`（空对象）的动画变换。与 `BoneAnimator` 类似，同样支持位移、旋转和缩放通道，但目标对象为空对象而非骨骼。

### 使用示例

```javascript
const nullAnim = new NullObjectAnimator(uuid, animation);
nullAnim.addKeyframe({
    channel: 'position',
    time: 0.0,
    data_points: [{ x: 0, y: 0, z: 0 }]
});
nullAnim.addKeyframe({
    channel: 'position',
    time: 2.0,
    data_points: [{ x: 10, y: 5, z: 0 }]
});
```

## 10. EffectAnimator 类 — 效果动画器

`EffectAnimator` 继承自 `GeneralAnimator`，用于管理粒子发射、声音播放等效果的时间轴触发。与变换动画器不同，效果动画器在特定时间点触发事件，而非连续变换。

### 功能说明

效果动画器通常配合关键帧的 `'effect'` 通道使用，在指定时间触发以下效果：
- **粒子效果**：在模型指定位置生成粒子
- **声音效果**：播放指定音效
- **自定义事件**：触发插件自定义逻辑

### 使用示例

```javascript
const effectAnim = new EffectAnimator(animation);

// 在 0.0 秒触发粒子效果
effectAnim.addKeyframe({
    channel: 'particle',
    time: 0.0,
    data_points: [{ x: 0, y: 0, z: 0 }]
});

// 在 0.5 秒播放脚步声
effectAnim.addKeyframe({
    channel: 'sound',
    time: 0.5,
    data_points: [{ x: 0, y: 0, z: 0 }]
});
```

## 11. Timeline 命名空间 — 时间轴

`Timeline` 是 Blockbench 的全局时间轴命名空间，管理动画播放、时间控制和关键帧选择。它是最常使用的动画系统入口。

### 属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `Timeline.time` | `number` | 当前时间轴时间，单位：秒 |
| `Timeline.playing` | `boolean` | 时间轴是否正在播放 |
| `Timeline.playback_speed` | `number` | 播放速度倍率 |
| `Timeline.animators` | `GeneralAnimator[]` | 当前激活的动画器列表 |
| `Timeline.selected` | `_Keyframe[]` | 当前选中的关键帧数组 |

### 方法

#### **`Timeline.setTime(time, editing?)`**
设置时间轴到指定时间位置。`editing` 为 `true` 时，标记为编辑状态（会触发关键帧值更新）。

```javascript
// 跳转到 1.5 秒位置
Timeline.setTime(1.5);

// 编辑模式跳转（正在拖动时间轴）
Timeline.setTime(2.0, true);
```

#### **`Timeline.revealTime(time)`**
滚动时间轴视图，使指定时间位置可见（不改变当前时间）。

```javascript
// 滚动到 3.0 秒位置
Timeline.revealTime(3.0);
```

#### **`Timeline.snapTime(time, animation?)`**
将时间对齐到吸附步长。根据当前动画的 `snapping` 设置，将时间值吸附到最近的帧位置。返回吸附后的时间值。

```javascript
// 假设当前动画 snapping = 24（每帧 1/24 ≈ 0.0417 秒）
const snapped = Timeline.snapTime(0.55);
// 返回: 0.5417（最接近的帧位置）
```

#### **`Timeline.getStep()`**
获取当前的对齐步长（秒），基于当前动画的 `snapping` 值计算。

```javascript
const step = Timeline.getStep();
// 若 snapping = 24，则 step = 1/24 ≈ 0.0417
```

#### **`Timeline.start()`**
开始播放时间轴。从当前时间位置开始播放。

```javascript
// 开始播放
Timeline.start();
```

#### **`Timeline.pause()`**
暂停时间轴播放。

```javascript
// 暂停播放
Timeline.pause();
```

#### **`Timeline.stop()`**
停止播放并重置时间到起点。

```javascript
Timeline.stop();
```

#### **`Timeline.unselect()`**
取消所有关键帧的选中状态。

```javascript
Timeline.unselect();
```

### TimelineMarker 接口

时间轴标记，用于在时间轴上标注特殊位置（如事件触发点、节拍点等）。

| 属性 | 类型 | 说明 |
|------|------|------|
| `color` | `number` | 标记颜色值 |
| `time` | `number` | 标记在时间轴上的位置，单位：秒 |
| `name` | `string` | 标记名称 |
| `uuid` | `UUID` | 标记唯一标识符 |

```javascript
// 创建时间轴标记
const marker = new TimelineMarker({
    color: 0xff6600,
    time: 1.0,
    name: '事件点'
});

// 添加到动画
animation.markers.push(marker);
```

### 完整示例

```javascript
// 动画播放控制综合示例
function playSelectedAnimation() {
    const anim = Project.animations.find(a => a.selected);
    if (!anim) return;

    // 跳转到动画起始位置
    Timeline.setTime(0);

    // 开始播放
    Timeline.start();

    // 播放状态监控
    const interval = setInterval(() => {
        if (!Timeline.playing) {
            clearInterval(interval);
            console.log('播放完成');
            return;
        }

        // 每帧检查是否超出动画长度
        if (Timeline.time >= anim.length) {
            if (anim.loop === 'once') {
                Timeline.pause();
            }
        }
    }, 16); // 约 60fps
}
```
